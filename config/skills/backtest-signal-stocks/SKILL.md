# Backtest Signal Stocks

## 概述

本 Skill 用于把研究信号（每日信息、紧缺度、电话会、专题研报）中提到的股票清单，映射到真实行情上，计算区间收益并生成复盘报告。核心目标不是算精确收益，而是回答：

1. 哪些信号真正赚到了钱？
2. 赚钱标的有无共同结构？
3. 下一阶段该把仓位调到哪些类型？

## 触发场景

- 用户给出一段时间范围和一组研究文件，要求"回测这些标的表现"
- 用户想复盘"每日信息/每日紧缺度/电话会/专题研报"选出股票的命中率
- 用户要求"从第一性原理"提炼赚钱框架

## 工作流

### Step 1：提取标的清单

从用户指定的研究文件或目录中提取 6 位 A 股代码。注意：
- 300/301/688/603/600/000/002 开头为常见代码
- 同一只标的可能在不同文件中重复出现，去重
- 记录每只标的出现过的信号源（用于后续归因）

### Step 2：确定统计口径

默认口径：
- 买入价 = 起始日开盘价（假设信号前夜已决定，次日开盘执行）
- 卖出价 = 结束日收盘价
- 区间收益 = (结束日收盘价 - 起始日开盘价) / 起始日开盘价 × 100%

如用户另有要求，按用户口径。

### Step 3：批量获取行情

使用 MCP 工具 `mcp__tdx-connector__tdx_kline` 获取日线数据：
- 沪市 code 传 6 位，setcode=1
- 深市 code 传 6 位，setcode=0
- period=4（日线），wantNum 取覆盖区间所需条数（通常 20-60）

多只股票可并行调用。逐只解析返回 JSON，取出起始日 Open 和结束日 Close。

### Step 4：计算收益并排序

用脚本 `scripts/calc_returns.py` 计算并生成表格：

```bash
python scripts/calc_returns.py --input prices.json --output report.md
```

输入 `prices.json` 格式：
```json
{
  "start_date": "20260615",
  "end_date": "20260623",
  "stocks": [
    {"name": "中船特气", "code": "688146", "setcode": 1, "open": 30.5, "close": 38.4}
  ]
}
```

### Step 5：归因与框架输出

在报告中必须包含以下章节：
1. **全量收益排序表**（按收益降序）
2. **按信号源/主题分组**（6N材料、去日化设备、每日信息等）
3. **确定性 × 收益矩阵**（用 B/E/C/G 四维度粗略归类）
4. **Losers 归因**（为什么有逻辑没涨）
5. **第一性原理框架**（赚钱 = B 供给刚性 × E 证据累积 × G 预期差 × C 催化剂）
6. **下一步操作建议**（按当前价位和确定性给出持有/加仓/观望）

## 四因子评分规则

| 维度 | 核心问题 | 高 | 中 | 低 |
|:----:|:---------|:--:|:--:|:--:|
| B 瓶颈 | 供给是否刚性？ | 产能/认证/矿源受限 | 有壁垒但可扩产 | 供给充足 |
| E 证据 | 是否有真实变化？ | 涨价/库存低/订单增 | 数据好转但不够强 | 仅有预期 |
| G 预期差 | 股价是否已透支？ | 低位/少覆盖/少人聊 | 部分反映 | 已大涨/高拥挤 |
| C 催化 | 未来 1-4 周有无事件？ | 明确可追踪 | 有但时间不确定 | 无 |

仓位建议：高 B+E+C+G → 重仓；高 B 低 E → 观察；低 B 高 C → 小仓位博弈。

## 输出规范

- 报告写入 `深度研究/MMDD-MMDD/YYYY-MM-DD_回测_关键词.md`
- 完成后必须同步到 IMA 聚宝盆对应周文件夹
- 关键数字用 🔥 标记 Top 10，用 🟡/🔴 标记确定性变化

## 常见坑

- 不要把 "区间涨跌" 字段直接当 6.15→6.23 收益，工具返回的区间涨跌可能是 20 日或当前相对某个参考点
- 300398 是飞凯材料，不是联瑞新材（688300），代码容易混淆
- 大市值平台型标的（北方华创等）确定性高但弹性低，要单独说明
- 股价涨幅不等于投资价值，必须结合利润映射空间判断
