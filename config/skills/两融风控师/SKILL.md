# 两融风控师 Skill v1.0

**定位**：个股融资盘压力的量化诊断器。不接买入/卖出建议，只输出事实、风险价位与级联压力。

**核心原则**：
1. 维保比例严格按交易所定义计算，**不混淆折算率与维保比例**。
2. 数据按锅师信息源体系分级调用，T0/T1优先，T2/T3交叉验证，T4仅作线索。
3. 计算假设必须显性化：1:1杠杆、忽略利息、静态折算率。实际规则以券商为准。

用户明确或隐含意图涉及以下任一，即调用本 Skill：
- 个股融资盘风险诊断
- 融资余额/融券余量分析
- 维持担保比例、爆仓线、追保线计算
- 个股两融数据查询与风控报告

## 三、核心计算

### 3.1 关键概念校准

**维持担保比例（维保比例）** 按交易所定义：

```
维保比例 = (现金 + 信用证券账户内证券市值总和) / (融资买入金额 + 融券卖出证券数量 × 当前市价 + 利息及费用)
```

**注意**：折算率 **不直接乘在维保比例分子上**。折算率影响的是「保证金可用余额」和「可融资金额」，而非维保比例本身。本 Skill 输出的是简化版维保比例，忽略利息和融券负债。

**简化 1:1 杠杆假设**：
- 自有资金 = 融资额 = F
- 初始买入总市值 = 2F
- 成本价 C 对应总持仓成本
- 当前股价 P 时，持仓市值 = 2F × (P/C)
- 维保比例 = 持仓市值 / 融资额 = 2F × (P/C) / F = 2 × (P/C)

因此，给定目标维保比例 T，对应股价：

```
P_T = T × C / 2
```

各阈值对应的股价：
- 300% 提取线：P_300 = 1.5 × C
- 150% 警戒线：P_150 = 0.75 × C（较成本价跌 25%）
- 130% 追保线：P_130 = 0.65 × C（较成本价跌 35%）
- 115% 强平线：P_115 = 0.575 × C（较成本价跌 42.5%）

**跌幅计算公式**：

```
跌幅 = 1 - P_T / C
剩余跌幅 = 1 - P_now / P_T
```

### 3.2 近3日融资买入额加权均价（C_weighted）

对近3日（指定日及前2个交易日）数据：

```
C_weighted = Σ(融资买入额_i × 当日均价_i) / Σ(融资买入额_i)
```

其中：
- 融资买入额_i：第 i 日融资买入金额
- 当日均价_i：第 i 日股票均价（优先用 tdx 均价，否则 成交额/成交量）

**C_high**：近3日中「融资买入额 × 当日均价」最大那一天的均价，代表近3日最高成本入场点。

**C_high_date**：C_high 对应的日期。

### 3.3 当前股价定位

- 近3日最高价 H_max、最低价 L_min
- 当前价 P_now 相对区间位置：
  - 位置 = (P_now - L_min) / (H_max - L_min) × 100%
  - >70%：上沿
  - 30%-70%：中位
  - <30%：下沿
- 相对 C_weighted 偏离：`(P_now - C_weighted) / C_weighted × 100%`

### 3.4 风控线倒推（基于1:1杠杆）

对 C_weighted 和 C_high 两个成本基准，分别计算：

| 目标维保比例 | 对应股价公式 | 跌幅 | 剩余跌幅（相对当前价） |
|:---|:---|:---|:---|
| 150% | P_150 = 0.75 × C | 25% | (P_now - P_150) / P_now × 100% |
| 130% | P_130 = 0.65 × C | 35% | (P_now - P_130) / P_now × 100% |
| 115% | P_115 = 0.575 × C | 42.5% | (P_now - P_115) / P_now × 100% |

**注意**：若 P_now < P_130，则剩余跌幅为负，表示已触发追保或强平。

### 3.5 极端情景：C_high 入场者当前状态

假设某融资客在 C_high 日满仓融资买入（1:1杠杆，即全部可用保证金买入）：

```
V_current = 2 × P_now / C_high
```

- 若 V_current ≥ 150%：安全
- 若 130% ≤ V_current < 150%：警戒区
- 若 115% ≤ V_current < 130%：已触发追保
- 若 V_current < 115%：已进入即时强平区间

**追保需补金额**（按原始融资额 F 的百分比）：

追保目标通常是将维保比例补至 140%。设需补金额为 M，则：

```
(Portfolio_value + M) / F = 1.40
M = 1.40 × F - Portfolio_value
```

Portfolio_value = 2F × P_now / C_high，因此：

```
M / F = 1.40 - 2 × P_now / C_high
```

### 3.6 密集爆仓区与踩踏烈度

- **一级爆仓区**：P_130 价位附近（触发追保，可能引发主动卖出或补保失败后被强平）
- **二级爆仓区**：P_115 价位附近（直接强平，无商量）
- **踩踏烈度**：

```
F_ratio = 融资余额 / 流通市值 × 100%
```

评级：
- < 3%：低烈度
- 3% - 8%：中等烈度
- > 8%：高烈度（需警惕级联强平）

**强平抛压粗略估算**：

```
Estimated_forced_selling = 融资余额 × 进入爆仓区概率因子
```

进入爆仓区概率因子：可简化为「当前价已跌破 P_130 则取 1，位于 P_130-P_115 之间取 0.5，未触及取 0」。

### 3.7 新增：融资余额动能与历史分位

- **融资余额变化率（近N日）**：
  ```
  F_change_N = (融资余额_T - 融资余额_T-N) / 融资余额_T-N × 100%
  ```
  建议计算 N=5、N=10、N=20。

- **融资余额占流通市值比历史分位**：
  若无法获取长周期数据，用近60日或今年以来数据估算。标注数据限制。

- **偿还/买入比**：
  ```
  Repay_buy_ratio = 融资偿还额 / 融资买入额
  ```
  >1：偿还压力大；<1：融资加仓。

## 五、专业补充：你可能没想到的维度

### 5.1 板块与标的特殊性

| 板块 | 特殊规则 | 对风控的影响 |
|:---|:---|:---|
| 科创板 | 涨跌幅20%，融资保证金比例可能上调 | 波动大，维保比例变化更快；券商可能要求更高保证金 |
| 创业板 | 涨跌幅20%，部分标的折算率偏低 | 同上 |
| 北交所 | 两融标的范围有限，流动性较低 | 强平冲击更大，但融资盘通常较少 |
| ST/*ST | 通常不可作为融资标的，折算率0 | 若已持仓，无法新增融资，只能平仓 |
| 沪深300 | 折算率最高，流动性最好 | 融资盘相对安全，但踩踏烈度仍取决于余额 |

### 5.2 集中度控制（券商内部风控）

部分券商对单票持仓集中度设限：
- 维保比例 150%-180%：单票集中度可能≤35%
- 维保比例 130%-150%：单票集中度可能≤25%
- 维保比例 <130%：可能强制要求减仓

本 Skill 不计算集中度，但输出中应**提醒**用户：若该股占两融账户比例过高，即使未触发全仓强平，也可能触发券商集中度限制而被要求减仓。

### 5.3 利息与费用的隐性侵蚀

融资利息按日计息（通常年化 6%-8%）。长期持仓会缓慢推升负债，降低维保比例。输出中可注明：
> 若融资客已持仓超过 [N] 日，实际维保比例略低于本报告简化值，需将利息纳入负债估算。

### 5.4 折算率下调的突袭风险

券商可能在股价大跌或基本面恶化时**临时下调**某标的折算率。这会导致：
- 保证金可用余额骤降
- 新增融资受限
- 已接近警戒线的账户被迫追加保证金

输出中应监控近30日内是否有券商/交易所下调该标的折算率。

### 5.5 融券余额与反向压力

融券余额高意味着空头力量强。当股价下跌时，空头获利了结可能加速反弹；但融券余量持续增加则加大下行压力。输出中可简评：
- 融券余额/融资余额比值 > 0.3：空头较活跃，需警惕双向挤压

### 5.6 融资盘与流通盘的错配

高融资余额占比（>8%）并不意味着必然踩踏。需结合：
- 大股东持股比例（若大股东占比高，实际流通盘更小，融资盘占比被低估）
- 近5日日均成交额（若成交额小，同样融资余额的平仓冲击更大）

建议输出：
```
融资余额 / 近5日日均成交额 = X 天（表示平仓所需交易日数，仅作参考）
```

## 六、执行流程

当用户请求两融诊断时：

1. **解析输入**：提取股票代码/名称、日期。若日期非交易日，取最近交易日。
2. **采集数据**：
   - 调用 tdx-connector 获取行情、股本、板块

   - 调用 IMA 聚宝盆搜索历史两融相关沉淀
3. **计算**：按第三章公式计算 C_weighted、C_high、风控线、维保比例、踩踏烈度等。
4. **生成报告**：按第四章输出格式生成结构化报告。
5. **信源标注**：在报告中标注主要数据信源等级（T2为主，T1/T3为辅）。
6. **风险提示**：强调简化假设、实际券商差异、时效性。
