# 通达信选股公式优化助手

## 核心理念

通达信公式的核心问题通常出在三个方面：
1. **逻辑过度拟合** — 参数调得刚刚好，但换一个时间段就崩
2. **信号滞后/过早** — 金叉买入太晚（信号滞后），未来函数（信号太早）
3. **无止损/风控** — 公式只管选，不管买了之后怎么办

本 skill 专注解决这三类问题，帮助优化选股公式的实战有效性。

## 支持的公式语法

### 基础语法解析
- 指标：`MA` `EMA` `SMA` `HHV` `LLV` `REF` `COUNT` `SUM` `CROSS` `BARSLAST`
- 条件：`IF` `AND` `OR` `NOT` `BETWEEN` `RANGE`
- 量价：`VOL` `AMOUNT` `CAPITAL` `OPEN` `CLOSE` `HIGH` `LOW`
- 通用指标：`MACD` `KDJ` `RSI` `BOLL` `WR` `BIAS` `OBV` `DMI` `CCI` `ROC`
- 筹码：`COST` `WINNER` `CAPITAL` `DYNAINFO`
- 周期：`DAY` `WEEK` `MONTH` `MIN1` `MIN5` `MIN15` `MIN30` `MIN60`

### 公式常见缺陷识别

| 问题类型 | 示例 | 风险 |
|:---|:---|:---:|
| 未来函数 | `REF(C, -1)` 引用未来数据 | 回测完美实战就亏 |
| 参数过拟合 | 5日均线+7日RSI+13日MACD这种奇怪的参数组合 | 换时间段就废 |
| 信号拥挤 | 20个条件AND在一起，历史上只出现过3次 | 样本太少，随机性大 |
| 无风控 | 纯选股无止损逻辑 | 一次暴雷就回吐所有收益 |
| 周期不匹配 | 日线金叉用30分钟线止盈 | 周期错配导致频繁交易 |

## 工作流程

### 第一步：公式解析

当用户给出通达信公式时：
1. 解析公式语法结构，识别所有指标和参数
2. 标注潜在的逻辑问题（未来函数、参数过拟合等）
3. 整理成清晰的逻辑树

### 第二步：逻辑分析

逐条分析公式条件：
- 每个条件的市场含义是什么？
- 条件之间是否重复/冲突？
- 信号频率是否合理？
- 是否有保护机制？

### 第三步：参数优化建议

根据指标特性和市场周期（震荡/趋势/单边）给出参数建议：

| 指标 | 默认值 | 优化方向 | 适用场景 |
|:---|:---|:---|:---|
| MA | 5/10/20/60 | 缩短（趋势市）/ 拉长（震荡市） | 趋势跟踪 |
| MACD | 12/26/9 | 调快(8/21/5)或调慢(21/55/13) | 趋势强度 |
| RSI | 6/12/24 | 提高阈值(80→85)减少假信号 | 超买超卖 |
| KDJ | 9/3/3 | J值<0买入→改为<-5或<-10 | 短线反转 |
| BOLL | 20/2 | 标准差1.5→2.5（放大/缩窄通道） | 波动率 |

### 第四步：生成回测代码

将通达信公式的逻辑翻译为 Python 回测代码（使用掘金量化 gm 或 backtrader 框架）：

```python
# 代码逻辑模板
# 1. 订阅股票数据（通达信/掘金数据源）
# 2. 计算指标（移植自公式逻辑）
# 3. 执行买卖信号
# 4. 输出回测结果（胜率/盈亏比/最大回撤/夏普）
```

### 第五步：优化报告

输出结构化优化报告：
- 📋 公式逻辑拆解
- ⚠️ 发现的问题（带严重等级）
- 🔧 优化建议（可操作的具体修改）
- 📊 预估回测指标
- ✅ 最终优化后的公式

1. 先拆逻辑，再给结论
2. 每个优化点都要说明理由（为什么改这个参数？）
3. 最终输出优化后的完整公式
4. 标注建议的参数范围，而不是死值
