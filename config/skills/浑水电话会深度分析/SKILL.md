# 浑水电话会深度分析 Skill

将浑水调研每日电话会总结，固化为标准化的四步投研管线——记录→瓶颈→景气度→信源验证→**IMA「聚宝盆」知识库（主输出）**。

## 核心理念

机构电话会是最接近产业真相的一手信息源。卖方研报有时滞、社群观点有噪音，但电话会里的买方提问和卖方回答，能直接反映当前市场最关心的供需矛盾与定价权变化。本skill确保每一份电话会都不被浪费。

## 触发条件

当用户发送浑水调研电话会总结文本时激活，典型格式：
- 「今天浑水电话会讲了XXX」
- 「昨天电话会总结：XXX」
- 用户直接粘贴电话会文字稿/要点

## 工作流程

### 阶段一：接收与解析

从用户发送的文本中提取结构化信息：

```
## 会议元数据
- 日期：
- 涉及行业/公司：
- 核心议题：

## 关键信息点
1. [供需变化]
2. [价格信号]
3. [产能扩张/收缩]
4. [买方关注焦点]
```

### 阶段二：产业瓶颈分析

对每个涉及的行业/环节，调用「产业链瓶颈投研skill」的方法论：

1. **卡点定位**：该环节在整个产业链中的不可替代性
2. **供给刚性判断**：产能扩张周期（6个月/12个月/24个月以上？）
3. **需求增速验证**：下游需求是否在加速/放缓
4. **价格传导**：上游涨价是否能向下游传导

🔴 **CHECKPOINT**：每个行业/环节四项分析全部覆盖后再进入阶段三。

### 阶段三：景气度扫描

调用「产业链瓶颈投研skill」步骤4b的方法论（已融合景气度扫描框架）：

1. **景气度方向**：上行/持平/下行
2. **供给端信号**：产能利用率、库存水平、扩产计划
3. **需求端信号**：订单可见度、下游扩产、客户集中度变化
4. **紧缺度评分**（1-5分）

### 阶段四：信源交叉验证

调用「信源检查skill」的分级标准：

**采集优先级**（详见 `G:\锅师\.workbuddy\memory\信源权重.md`）：

| 优先级 | 信源 | 操作 |
|:---:|:---|:---|
| **P0.5** | 聚宝盆（内部缓存） | 历史同类分析引用+对比——知道昨天判断了什么，今天只做增量 |

**信源分级**（按内容实质，不以渠道代内容）：
| 等级 | 定义 | 操作 |
|------|------|------|
| **A** | 法定披露 + 可溯源原文交叉验证一致 | 直接采信，标注「原文可查看[浑水调研]」 |
| **B** | 电话会主观判断 + 行业数据佐证 | 标注"参考"，浑水调研来源同级优先 |
| **C** | 电话会单一判断 + 无其他信源 | 标注"待验证" |
| **D** | 雪球/股吧/微信群 | 禁止引用 |

对每个关键结论标注信源等级。当出现A级矛盾（两个A级信源互相矛盾）→ 暂停输出，请求用户确认。

🛑 **STOP**：所有结论必须≥C级才能进入阶段五。D/E级存在→暂停，向用户报告。

## S级信号判定标准

满足以下任一条件即为S级：

| 信号 | 判断标准 |
|------|----------|
| **供给断崖** | 某环节产能收缩30%+ 且扩产周期>18个月 |
| **定价权突变** | 某环节成功提价20%+ 且下游接受 |
| **需求跳升** | 下游需求增速环比翻倍（电话会多方交叉确认） |
| **产业拐点** | 连续3次电话会景气度方向一致反转 |

S级信号产出：已进入IMA笔记（自动） + 独立深度报告（额外生成）

## IMA同步技术规范（2026-06-14 沉淀）

### 文件命名

统一格式：`YYYY-MM-DD_电话会日报.md`（固定后缀，与 `YYYY-MM-DD_紧缺度日报.md` 形成区分），严禁 `电话会_YYYY-MM-DD_...` 倒置格式或其他变体。

### 阶段五 IMA沉淀执行细则

**步骤顺序**：先写IMA笔记（5A），再写本地md文件（5B），最后同步md文件到IMA（5C）。

#### 5A. 创建笔记并关联知识库

Python直接调用IMA API（header: `ima-openapi-clientid` / `ima-openapi-apikey`，非 X-IMA 前缀），两步完成：

1. `POST /openapi/note/v1/import_doc` → 获得 `note_id`
2. `POST /openapi/wiki/v1/add_knowledge`（`media_type=11`，`note_info.content_id=<note_id>`，**必须带 `folder_id`**）→ 关联到聚宝盆

#### 5C. 同步md文件到IMA

笔记关联后，将本地分析md文件同步上传至同一文件夹：

1. `POST /openapi/wiki/v1/create_media` → 获得 `media_id` + COS临时凭证
2. COS PUT上传（sha1签名 + `x-cos-security-token`，content_type 用 preflight 返回值）
3. `POST /openapi/wiki/v1/add_knowledge`（`media_type=7`，**必须带 `folder_id`）

### 🛡 IMA同步异常处理

| 步骤 | 失败场景 | 一线修复 | 仍失败兜底 |
|------|---------|---------|-----------|
| import_doc | 401/网络超时 | 检查凭证，重试1次 | 跳过IMA，本地文件正常写入 |
| add_knowledge | folder_id无效 | 搜索周文件夹名获取正确ID | 先 `create_folder` 创建，再关联 |
| create_media | invalid media_type | 用 preflight 脚本返回值，不用硬编码 | 跳过文件上传，笔记已同步 |
| COS PUT | 403签名错误 | 用 cos-upload.cjs 脚本代替自签 | 记录 media_id，手动上传 |
| 自检缺文件 | 本地有云端无 | 立即补传（create→COS→add） | 告警输出 ⚠️ 缺N个 |

## 紧缺度日报快速采集协议 v2.0（2026-06-18 升级）

### 核心升级

v1.0（搜标题）→ v2.0（读全文+NLP提取）：

| 维度 | v1.0 | v2.0 |
|:---|:---|:---|
| 采集精度 | 标题关键词匹配 | 正文全文NLP提取 |
| 信号捕获 | 标题写了才算 | 正文埋藏的也能抓到 |
| Token管控 | 低（只读标题） | 可控（条数上限+token预算） |
| 可提取字段 | 仅关键词 | 价格/产能/时间/客户/供应链事件 |

### 采集流程

```
步骤1：日期预过滤（2-4次API）
  ├─ 搜索 query="YYYYMMDD"（如 20260617）→ 翻页至 is_end=true
  └─ 搜索 query="MMDD"（如 0617）→ 1页即可
  └─ 合并去重（按 media_id）→ 得到候选列表

步骤2：正文读取 + NLP提取（0次额外API，利用S1返回的content）
  ├─ 从 search_knowledge 返回的 info_list 逐条读取 content 字段
  ├─ 对每条正文执行结构化提取：
  │   ├─ 价格信号：数字+单位（元/吨、$/片、元/公斤等）+ 涨跌幅
