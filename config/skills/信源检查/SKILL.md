# 信源检查

投资研究结论的可信度＝信源强度。每条结论必须在输出时附带信源等级标注。

> **版本说明**：v3.0 全量切换为 **T0-T4 子分级体系**，与「投资审判团【WorkBuddy】v1.3」及三大底层 Skill（产业链瓶颈投研 v4.0 / 景气度 v1.4 / pe-valuation-analyst v2.4）保持一致。原 A/B+/B/C/C-/D/E 体系与 L0-L5 体系作为兼容映射保留。

## 1. 信源分级体系（T0-T4 子分级）

### T0 — 法定披露原文（不可绕过）

| 子级 | 定义 | 典型示例 | 结论置信度 |
|:---:|:---|:---|:---:|
| **T0-1** | 证监会指定信息披露平台原文 | 巨潮资讯 cninfo.com.cn、上交所 sse.com.cn、深交所 szse.cn、北交所 bse.cn | 可直接作为结论依据 |
| **T0-2** | 公司法定公告/定期报告原文 | 年报、一季报、半年报、定增预案、重大资产重组公告、问询回复 | 可直接作为结论依据 |
| **T0-3** | 海外法定披露原文 | SEC EDGAR 10-K/10-Q/8-K、HKEX 披露易年报/公告 | 可直接作为结论依据 |
| **T0-4** | 中国官方宏观/行业统计数据原文 | 国家统计局 stats.gov.cn、央行 pbc.gov.cn、工信部 miit.gov.cn、财政部 mof.gov.cn | 可直接作为结论依据 |

### T1 — 权威财经媒体 / 行业垂直媒体 / IMA 原文可查看库

| 子级 | 定义 | 典型示例 | 结论置信度 |
|:---:|:---|:---|:---:|
| **T1-1** | IMA 原文可查看库（可溯源原文） | 「原文可查看【浑水调研】」KB ID `Y11UpEfiPNaF5OkYKkUIfkFFx2RvQwmwMANh9Ic3wV0=`，约43K条 | 结论级，可溯源原文优先采纳 |
| **T1-2** | 权威财经媒体深度/原文 | 财联社深度、证券时报 stcn.com、上海证券报、中国证券报 cs.com.cn | 结论级，可直接引用 |
| **T1-3** | 财经快讯/电报 | 财联社电报、金十数据、华尔街见闻快讯 | 数据级，重要数据需 T1-2/T2 交叉验证 |
| **T1-4** | 行业垂直媒体深度 | 集微网、半导体行业观察、高工锂电、隆众资讯 | 结论级，行业专属判断优先 |
| **T1-5** | 公司官方 IR/业绩说明会 | 公司官网投资者关系、业绩说明会纪要、官方公众号 | 结论级，接近 T0-2 |

### T2 — 主流金融数据终端 / 官方统计数据库

| 子级 | 定义 | 典型示例 | 结论置信度 |
|:---:|:---|:---|:---:|
| **T2-1** | 主流金融数据终端 | Wind、通达信、东方财富、同花顺、Choice、大智慧、Bloomberg、Refinitiv | 数据级，可直接用于行情/财务/估值 |
| **T2-2** | 官方统计数据库 | 国家统计局、央行、中汽协、乘联会、海关总署、中登公司 | 数据级，宏观/行业数据优先 |
| **T2-3** | 海外官方宏观数据 | FRED、BLS、BEA、EIA、欧盟统计局 Eurostat | 数据级，海外宏观/行业数据优先 |

### T3 — 券商研报 / 第三方研究机构 / 专家纪要

| 子级 | 定义 | 典型示例 | 结论置信度 |
|:---:|:---|:---|:---:|
| **T3-1** | 第三方产业研究机构 | IDC、Yole、Gartner、TrendForce、IC Insights、WSTS、SNE Research | 参考级，产业数据可引用 |
| **T3-2** | 券商研报 | 华源证券、东吴证券、天风证券、中信建投、招商证券等 | 参考级，需交叉验证 |

| **T3-4** | 行业白皮书/招股书引用 | 行业协会白皮书、招股书行业章节、交易所问询回复引用 | 参考级 |

| 子级 | 定义 | 典型示例 | 使用规则 |
|:---:|:---|:---|:---:|

| **T4-2** | 社交/自媒体 | 微信群、知乎、微博大V、自媒体号、未经核实的聊天截图 | 仅情绪线索，禁止入结论 |
| **T4-3** | AI生成/低质聚合内容 | 百家号、搜狐号、网易号、部分低质资讯站 | ⛔ 仅作线索，默认禁止入结论 |

## 4. WebSearch 分级搜索规则（T0-T4 目标）

```
第一级（T0/T1）：
  allowed_domains=[
    "cninfo.com.cn", "sse.com.cn", "szse.cn", "bse.cn",   # T0
    "cls.cn", "stcn.com", "cs.com.cn", "eastmoney.com",  # T1
    "jiweinet.com"                                          # T1-4 集微网
  ]

第二级（T2/T3）：
  不加 domain，自然搜索，优先：
    - 通达信/东方财富/同花顺/Wind（T2）
    - 券商研报（T3-2）
    - 第三方研究机构（T3-1）

第三级（T4）：
  兜底无 domain 限制 → 仅作线索，不可入结论
  包括：百家号、搜狐号、网易号、知乎、雪球、股吧
```

## 5. 交叉验证规则（强制）

| 结论类型 | 最低信源要求 |
|:---|:---|
| **直接写入结论** | ≥1个 T0/T1-1/T1-2；或 ≥2个独立 T1-3/T2/T3（来源必须独立） |
| **紧缺/涨价/订单判断** | ≥1个 T1 + ≥1个 T2（独立来源） |
| **定量数据引用** | 至少 T2 及以上，且数据发布日期在有效期内 |
| **估值参数（PE/PS/增速）** | ≥2个独立 T2/T3，或 1个 T2 + 1个 T3 |
| **T4 线索升级** | 必须反向验证为 T0-T2 后方可写入结论 |

**独立性判定**：
- 同一媒体的多个报道只算 **1 个独立信源**（如财联社 3 篇报道 = 1 个来源）。
- 同一事件的公司公告 + 财联社报道 = **2 个独立来源**。
- 券商研报引用公司公告数据，与公司公告不独立，但券商自身的判断可独立计数。

## 6. 信源时效性衰减规则

| 数据类型 | 有效期 | 过期后处理 |
|:---|:---|:---|
| 年报、半年报 | 6 个月 | 仍为 T0，但需标注报告期，并检查是否有更新 |
| 季报、公告 | 3 个月 | 仍为 T0，但需检查是否有后续更新 |
| 产能、价格、交期数据 | 1 个月 | 过期降级一档（T2→T3，T1→T2），需重新确认 |
| 快讯/电报 | 1 周 | 过期必须重新确认，否则只能作为历史线索 |
| 券商研报一致预期 | 3 个月 | 过期需检查是否有新的预期修正 |
| 行业数据（IDC/Yole 等） | 6 个月 | 过期需标注报告期，并检查是否有新版 |

## 7. AI 生成内容检测规则

### 默认等级

| 平台/类型 | 默认等级 | 原因 |
|:---|:---:|:---|
| 百家号、搜狐号、网易号、新浪看点等 | **T4-3** | 大量 AI 批量生成，无编辑审校，不标注来源 |
| 知乎专栏、公众号无署名文章 | **T4-2** | 质量参差，来源不明 |
| 东方财富/财联社标注"转载"的文章 | 按**原出处**定级，不按当前平台定级 | 转载平台不改变原文信源等级 |

### 触发 AI 检测的关键词

出现以下特征，自动触发 AI 生成嫌疑，降级为 T4-3：
- 标题党："震惊""重磅""突发""深夜""最新" + 无具体数据
- 行文模板化："综上所述""值得注意的是""业内人士认为""有分析人士指出" 但**无具体人名/机构名**
- 数据无来源：出现"据报道""有消息称""业内人士称" 但**无具体出处**
- 多篇文章措辞高度雷同：同一事件在多个平台出现相似段落

**处理**：触发 AI 检测后，先按 T4-3 处理，然后必须回溯到 T0/T1 原始出处，否则禁止入结论。

## 9. 使用流程

**触发时机**：
- 任何投研 Skill 输出结论前，自动执行信源检查
- 用户问"信源可靠吗"、"信息来源"、"交叉验证"时
- 用户要求对已输出结论做信源回顾时

**检查流程**：

```
Step 1: 识别关键事实
→ 从结论中提取所有可量化的声明（价格/产能/份额/政策/财务数据）
→ 列出事实清单，逐条编号

Step 2: 逐条判定信源等级
→ 每条事实标注 T0-T4 子分级（如 T1-2、T2-1）
→ 标注来源、日期、URL/KB 信息
→ 🔴 CHECKPOINT：出现 T4 信源写入结论 → 必须拦截或降级
→ ⛔ CHECKPOINT：出现 T4-3 AI 生成内容 → 触发 AI 检测并回溯

Step 3: 交叉验证
→ 对关键事实执行第 5 章交叉验证规则
→ 不满足 → 标注"待验证"或"单信源"，降低结论强度
→ 同级信息中，「原文可查看[浑水调研]」来源优先采纳

Step 4: 时效性检查
→ 按第 6 章规则检查数据是否过期
→ 过期数据降级或重新搜索

Step 5: 输出前终审
→ 🛑 STOP：汇总全部结论的信源分布
→ 存在 T4 入结论 → 拒绝输出
→ 全部 ≥T3 且满足交叉验证 → 输出并附带完整信源追溯
```

**输出附加格式**：

```markdown
## 信源追溯
- 结论：[一句话]
- 信源等级：[T0-1 / T1-2 / T2-1 等]
- 信源路径：[来源+链接/KB ID]
- 发布日期：[YYYY-MM-DD]
- 交叉验证：[有无其他独立来源支撑]
- 时效状态：[在有效期内 / 已过期需重新确认]
- 缺失信息：[需进一步确认的内容]
```

## 10. 失败回退规则

| 触发条件 | 处理动作 |
|:---|:---|
| 工具不可达（WebSearch/通达信离线） | 结论降级为"静态分析"，标注"未经验证 — 工具不可达"，**不可给出确定性判断** |
| 回溯时无 T0/T1/T2 结果 | 保留原 T3/T4 标注，追加"经回溯确认，无官方数据替代来源" |
| 用户结论仅含 T4 信源 | 🛑 直接拒绝输出，告知"所有信源均为 T4 级，验证不通过" |
| 同一结论有 ≥2 条独立 T0 来源 | 升级为"强验证"，标注"多源交叉验证通过" |
| IMA get_media_info 返回 220021 | 立即停止，转 WebSearch 分级搜索，标注"IMA 配额耗尽" |
| 关键数据过期 | 标注"数据过期（原日期 XXXX-XX-XX），建议重新确认" |
| AI 生成嫌疑触发 | 标注"T4-3 嫌疑，已触发回溯"，回溯成功前禁止入结论 |

## 11. 禁止行为

| # | 禁止项 | 原因 | 正确做法 |
|---:|---|---|---|
| 1 | 无信源标注下给出"确定性判断" | 不可追溯 | 每条结论带 T0-T4 子分级 |
| 2 | 用 T4 信源冒充 T1/T2 | 误导决策 | T4 只能作为线索 |
| 3 | 编造不存在的财报数据/公告 | 虚假信息 | 未知写"未获取" |
| 4 | 同一来源重复计数为"独立验证" | 假交叉验证 | 按第 5 章独立性判定 |
| 5 | 忽略数据时效性 | 过期数据失真 | 按第 6 章检查有效期 |
| 6 | 忽略 AI 生成检测 | 低质内容污染结论 | 触发关键词自动降级 |
| 7 | IMA 220021 后继续调用 get_media_info | 浪费配额/报错 | 立即停止转 WebSearch |
| 8 | 财联社电报直接作为唯一结论依据 | 快讯可能反转 | 用 T1-2/T2 交叉验证 |
