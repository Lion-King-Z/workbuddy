# 信源检查

投资研究结论的可信度＝信源强度。每条结论必须在输出时附带信源等级标注。

---

## 2. 信源强度分级规则

| 等级 | 定义 | 示例 | 结论置信度 |
|:---:|:---|:---|:---:|
| **A级** | 法定披露原文 | 公司公告、财报、SEC/HKEX 法定文件原文 | 可直接用作结论依据 |
| **B级** | 机构纪要/权威报告 | 机构电话会纪要（浑水调研可溯源原文 > 36氪转载）、统计局/央行/Wind/行业白皮书 | 可引用，标注来源 |
| **C级** | 专业媒体/分析师研报 | 财联社/证券时报/券商研报 | 需交叉验证 |
| **D级** | 社区讨论/推论 | 知乎/Reddit | 线索级别，不可直接引用 |
| **E级** | 未证实传闻 | 微信群/小作文 | 禁止引用 |

### 信源采集优先级

**IMA 知识库实际可读情况**（详见 `G:\锅师\.workbuddy\memory\信源权重.md`）：

| 优先级 | 信源 | 可读性 |
|:---:|:---|:---|
| **P0** | 「原文可查看【浑水调研】」 | 唯一可读原文的 IMA 知识库，优先检索 |
| P1 | 36氪/华尔街见闻 外资行纪要 | 公开渠道，精准过滤 |
| P2 | WebSearch + TrendForce/SEMI/Yole | 行业公开数据 |
| P2 | 「叫我第一名」「基业长青」 | 仅搜标题，作为线索→回 P0 找原文 |
| P3 | 深证信公告库 | 公告标题可搜，无原文可读 |
| P4 | 雪球/股吧/微信群 | 仅情绪线索 |

**规则**：「原文可查看【浑水调研】」中的内容定级 B（可溯源原文）而非自动 A。硬数据+交叉验证一致→升 A。同级信息浑水调研版本优先采纳。

## 3. 使用规则

**触发时机**：
- 任何投研 Skill（产业链瓶颈/国策主题/财报分析等）输出结论前，自动执行信源检查
- 用户问"信源可靠吗"、"信息来源"、"交叉验证"时
- 用户要求对已输出结论做信源回顾时

**检查流程**：

```
Step 1: 识别关键事实
→ 从结论中提取所有可量化的声明（价格/产能/份额/政策/财务数据）
→ 列出事实清单，逐条编号

Step 2: 逐条判定信源等级
→ 每条事实标注来源层次（L0-L5）和信源等级（A/B/C/D/E）
→ 🔴 CHECKPOINT：当出现 D/E 级结论时，必须暂停并向用户发出警告
→ 同级信息中，「原文可查看[浑水调研]」来源优先采纳（可溯源原文）

Step 3: 回溯升级
→ 不足 A/B 级的结论，尝试从 memory 或 WebSearch 回溯到 L0/L1
→ 🔴 CHECKPOINT：回溯成功→升级标注；回溯失败→保留原等级并标注"建议尽调确认"

Step 4: 输出前终审
→ 🛑 STOP：汇总全部结论的信源分布，查看是否存在 E 级
→ E 级存在 → 拒绝输出，告知用户："以下结论信源不可靠，已拦截：[列举]"
→ 全部 ≥D 级 → 输出并附带完整信源追溯
```

**输出附加格式**：
```
## 信源追溯
- 结论：[一句话]
- 信源等级：[A/B/C/D]
- 信源路径：[来源+链接]
- 交叉验证：[有无其他来源支撑]
- 缺失信息：[需进一步确认的内容]
```

## 4. 失败回退规则

由于信源检查依赖工具链（WebSearch / 通达信 / IMA），存在工具不可达或数据缺失的常见失败场景。以下是必须显式处理的三种失败路径：

| 触发条件 | 处理动作 |
|:---|:---|
| 工具不可达（WebSearch/通达信离线） | 结论降级为"静态分析"，标注"未经验证 — 工具不可达"，**不可给出确定性判断** |
| 回溯时无L0/L1结果 | 保留原C/D级标注，追加"经回溯确认，无官方数据替代来源" |
| 用户结论仅含E级信源 | 🛑 直接拒绝输出，告知"所有信源均为E级，验证不通过"，并提供替代建议（需核查的维度+推荐查询路径） |
| 同一结论有≥3条独立L0来源 | 自动升级为A级，标注"多源交叉验证通过" |

---

## 5. 禁止行为

- 无信源标注下给出"确定性判断"
- 用D/E级信源冒充A/B级
- 编造不存在的财报数据/公告
- 一篇来源不足时强行下结论

详见 `references/source-detail.md` 获取完整信源图谱。

## 6. 全局检查点规则

四步流程必须按顺序执行，不可跳过。🛑 STOP 触发时必须输出拦截信息，不继续后续步骤。
