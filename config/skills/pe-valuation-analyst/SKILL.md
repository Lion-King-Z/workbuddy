# pe-valuation-analyst

把一句市场叙事收口到**单只标的的 PE 估值落地**。

这是一条独立的方法论工作流：从一句市场故事出发，走完估值消化路径，用硬信源验证催化，做中枢加权估值，**叠加财务健康评估**，给出仓位纪律和验证时间线。最终产出一份像锋利的研究搭档在用大白话讲逻辑的报告。

**v2.5 核心升级**：新增 **Step 4.5 财务健康评估** — 在估值落地之前先扫描偿债能力、运营效率、现金流质量，输出财务风险标签，直接供窄门筛股等下游Skill使用。

## 核心承诺

给定一只标的（含代码）和它的产业链故事，跑完估值落地工作流，返回一个清晰的、信源可溯源的结论：

`市场故事 → 系统变化 → 估值消化路径 → 产业链卡位 → 财务健康评估 → 多档PE/PEG校验 → 中枢加权基准 → 催化双轨+概率加权 → 四档风险情景 → 仓位纪律+验证时间线`

答案读起来应该像一个冷静的研究伙伴在用正常语言讲清楚逻辑链。

## 默认行为

深度落地是默认。

当用户给出标的代码、公司名，或问"XX 现在估值怎么看 / 能不能买 / PE 打低路径 / 仓位建议 / 验证节点 / 财务健康"时，先跑估值落地工作流，再给最终结论。

依赖现价/财报/一致预期/公告/订单的信息，必须用实时源：tdx-connector（行情/一致预期/财务）、web/search（公告/研报/新闻）、ima-mcp（浑水调研原文库）。无法实时取数时，明确标注哪些事实需要核查，并给出确切核查路径。

## 请求路由

先分类请求，再进对应模式。

- **完整估值落地**：用户给标的 + 产业链故事。跑完整工作流，返回落地报告。
- **单点校验**：用户只问某一环（"PE 打低路径清楚吗"/"催化有哪些"/"中枢 PE 给多少合理"/"财务健康怎么样"）。只跑对应章节。
- **仓位复核**：用户已有估值判断，要落地仓位和止损。跑第九步仓位纪律 + 验证时间线。
- **估值对话**：用户想推演/讨论。用紧问题把判断逼向信源、卡位、证伪条件。

## 工作流 — PE 估值落地 v2.5

跑完整套，适用于完整估值落地。每一步都要带信源标注。

### 第 0 步：取数与信源分级

落地前必须取齐下列数据，并标注每一项来源：

| 数据项 | 首选源 | 信源标注 |
|--------|--------|---------|
| 现价/涨跌/市值/股本 | tdx-connector 行情 | [T2] |
| 静态/TTM EPS、最新财报 | tdx-connector 财务 | [T2] |
| 多档一致预期 EPS（N+1/N+2/N+3） | tdx-connector 一致预期 | [T3] |
| 资产负债表/现金流量表关键项 | tdx-connector 财务 / 财报附注 | [T2] |
| 产业链卡位/订单/客户结构 | ima-mcp 原文可查知识库 | [T1] 最高优先 |
| 研报深度推荐/行业格局 | 原文库 / 公开研报 | [T1]/[T3] |
| 公告/重大事项 | 公司公告 | [T3] |

**信源分级体系**：

- `[T1]` 原文可查（一手纪要/深度推荐）—— 可读原文，最高采集优先级
- `[T2]` Web搜索 / 行情 / 财报 —— 可直接核实
- `[T3]` 一致预期 / 卖方研报 —— 机构预测，需跟踪上修/下修趋势

**双重验证规则**：关键催化/卡位判断必须有 ≥2 个独立信源交叉验证，标注 `✅双重验证`；仅单信源标注 `（单信源）` 并提示风险。

### 第 1 步：决策摘要面板（结论先行）

顶部一张表，6 行内给完核心结论。这一步是整份报告的"电梯演讲"。

| 字段 | 内容 |
|------|------|
| 信号 | ↑↑强买入 / ↑买入 / →持有 / ↓减仓 / ↓↓清仓（一句话理由） |
| 核心逻辑 | 一句话讲清"什么驱动 → 什么兑现 → 什么重塑" |
| 基准估值 | 市值 + 对应 PE 倍数 + 对应年份净利 |
| 催化后估值 | 市值区间 + 催化增量合计 + 概率加权后增量 |
| 当前定性 | 一句话定位"当前价是对未来的什么折现" |
| 仓位建议 | 区间百分比 |

摘要下紧跟两行：**估值消化路径**一句话 + **最大风险**一句话。

### 第 2 步：估值消化路径（PE 打低时间轴）+ 🆕一致预期修正追踪

把静态高 PE 拆成多档预测 PE，标注每年消化百分比，回答"高估值多久能消化、靠什么消化"。

```
静态 XXXx → N+1E XXx → N+2E XXx → N+3E XXx（X年消化XX%）
```

要点：
- EPS 一致预期趋势要标（连续上修/下修/修复），趋势转好用 ✓
- 综合评级（机构覆盖家数 + 评分）标来源
- "PE 打低清晰度"给定性判断（极高/清晰/模糊/存疑）

**🆕 2.1 一致预期修正追踪（v2.6新增）**

追踪过去4周 N+1E/N+2E 一致预期的修正趋势——这是重要的领先信号。

| 修正指标 | 计算方式 | 判断 |
|:---|:---|:---|
| 修正次数（上修 vs 下修） | 过去4周上修次数 / 下修次数 | 上修>下修→趋势向好 |
| 修正幅度 | (最新N+1E - 4周前N+1E) / 4周前N+1E | >+5%→显著上修 / <-5%→显著下修 |
| 覆盖机构数变化 | 当前覆盖数 vs 4周前 | 新增覆盖→关注度上升 |

**修正趋势标签**：

| 修正趋势 | 条件 | 对G评分的影响 | 含义 |
|:---|:---|:---:|:---|
| 📈 强上修 | 上修次数≥3 且 幅度>+5% | G+1 | 分析师集体上调，行情启动前兆 |
| 📈 温和上修 | 上修>下修 且 幅度+2~5% | G+0.5 | 预期改善中 |
| → 稳定 | 上修≈下修 且 幅度-2~+2% | 不变 | 预期稳定 |
| 📉 温和下修 | 下修>上修 且 幅度-2~-5% | G-0.5 | 预期恶化中 |
| 📉 强下修 | 下修次数≥3 且 幅度<-5% | G-1 | 分析师集体下调，风险预警 |

**输出格式**：

| 指标 | N+1E | N+2E |
|:---|:---:|:---:|
| 4周前一致预期 | X.XX | X.XX |
| 当前一致预期 | X.XX | X.XX |
| 修正幅度 | +X% | +X% |
| 上修/下修次数 | X/X | X/X |
| 覆盖机构数 | X家（4周前X家） | X家 |
| 修正趋势 | 📈温和上修 | 📈温和上修 |

> 一致预期修正趋势直接影响G预期差评分。强上修=G+1，强下修=G-1。

### 第 4 步：产业链卡位图谱

把标的放进产业链层级，说清"它在哪一层、和谁竞争、壁垒是什么"。

| 层级 | 公司 | 核心产品 | 与标的的关系 |
|------|------|---------|------------|

层级从上到下：终端客户 → 一级供应商（总成方案商）→ 二级供应商（标的所在层）→ 同层竞争者。

图谱下用 2-3 段话说清：
- 标的的核心壁垒（不是"产品绝无仅有"，而是"几位一体的转换成本/联合开发周期/认证壁垒"）
- 最接近的竞争对手差在哪、差几个季度
- 是否会直接冲突（客户/区域错位）

### 第 5 步：🆕 财务健康评估（v2.5新增）

**定位**：在估值判断之前，先扫描标的财务安全性。财务风险不是估值，但会决定估值的安全边际和PE中枢的折扣。

**所需数据**（来自最新财报/年报）：

| 数据项 | 公式 | 说明 |
|:---|:---|:---|
| 总资产 | 资产负债表 | — |
| 总负债 | 资产负债表 | — |
| 流动资产 | 资产负债表 | — |
| 流动负债 | 资产负债表 | — |
| 速动资产 | 流动资产 - 存货 | — |
| 有息负债 | 短期借款+长期借款+应付债券+一年内到期非流动负债 | 核心信用指标 |
| 利息费用 | 利润表/现金流量表 | — |
| EBIT | 息税前利润 | 用于利息覆盖 |
| 营业收入 | 利润表 | — |
| 营业成本 | 利润表 | — |
| 平均存货 | (期初存货+期末存货)/2 | 用于周转天数 |
| 平均应收账款 | (期初应收+期末应收)/2 | 用于周转天数 |
| 净利润 | 利润表 | — |
| 经营活动现金流净额 | 现金流量表 | — |
| 资本开支 | 购建固定资产等支付的现金 | — |

**5.1 偿债能力（4项）**

| 指标 | 公式 | 健康阈值 | 风险标签 |
|:---|:---|:---:|:---|
| 资产负债率 | 总负债 / 总资产 | <60% | >70%→高杠杆风险；>80%→财务脆弱 |
| 流动比率 | 流动资产 / 流动负债 | >1.5 | <1.0→短债偿付压力大 |
| 速动比率 | 速动资产 / 流动负债 | >1.0 | <0.8→流动性紧张 |
| 利息覆盖倍数 | EBIT / 利息费用 | >5 | <3→偿债能力弱；<1→利息吃利润 |

**5.2 运营效率（3项）**

| 指标 | 公式 | 健康阈值 | 风险标签 |
|:---|:---|:---:|:---|
| 存货周转天数 | 平均存货 / 营业成本 × 365 | 行业低位 | 显著高于行业均值→库存积压或产品滞销 |
| 应收账款周转天数 | 平均应收账款 / 营业收入 × 365 | 行业低位 | 显著高于行业均值→回款能力弱、现金流被占用 |
| 总资产周转率 | 营业收入 / 平均总资产 | 行业均值 | 显著低于行业均值→资产利用效率低 |

**5.3 现金流质量（3项）**

| 指标 | 公式 | 健康阈值 | 风险标签 |
|:---|:---|:---:|:---|
| 经营现金流/净利润 | 经营现金流净额 / 净利润 | >1.0 | <0.7→利润含金量低；<0→经营现金流为负 |
| 自由现金流/营收 | (经营现金流净额 - 资本开支) / 营业收入 | >5% | 持续为负→扩张靠融资、内生造血弱 |
| 资本开支/经营现金流 | 资本开支 / 经营现金流净额 | <80% | >100%→资本开支过度、挤压现金流 |

**5.4 财务风险标签输出**

根据上述10项指标，输出标准化风险标签：

| 标签 | 触发条件 | 对估值的影响 |
|:---|:---|:---|
| 高杠杆风险 | 资产负债率>70% | PE中枢下调5-10% |
| 短债偿付压力 | 流动比率<1.0 或 速动比率<0.8 | 下调估值安全边际 |
| 利息覆盖不足 | 利息覆盖倍数<3 | 业绩对利率敏感，PE下调 |
| 利润含金量低 | 经营现金流/净利润<0.7 | 盈利质量打折，PE需折扣 |
| 现金流紧张 | 自由现金流/营收<0% | 内生增长能力弱，估值承压 |
| 库存积压 | 存货周转天数显著高于行业均值 | 存在减值/降价风险 |
| 回款能力差 | 应收周转天数显著高于行业均值 | 营运资金占用，增长质量差 |
| 资产效率低 | 总资产周转率显著低于行业均值 | 重资产低效，ROE承压 |
| 资本开支激进 | 资本开支/经营现金流>100% | 投资回报不确定，稀释现金流 |
| 财务健康 | 无上述异常 | 估值安全边际可维持 |

**输出格式**：

| 维度 | 指标 | 数值 | 行业参考 | 风险标签 |
|:---|:---|---:|---:|:---|
| 偿债能力 | 资产负债率 | 45% | 50% | 🟢 财务杠杆适中 |
| 偿债能力 | 流动比率 | 1.2 | 1.5 | 🟡 短债覆盖偏紧 |
| 运营效率 | 存货周转天数 | 120天 | 90天 | 🟡 库存周转偏慢 |
| 现金流质量 | 经营现金流/净利润 | 0.65 | 1.0 | 🟡 利润含金量偏低 |

> 财务风险标签必须在后续风险情景量化（第8步）和仓位纪律（第9步）中被引用。

### 第 7 步：中枢加权估值（计算工卡）

这是基准估值的"算盘"。用 code 块呈现完整计算链。

```
中枢PE XXx（可比公司中位数：A XXx / B XXx / C XXx）
  × 品牌系数 X.X（B2B 非终端品牌=1.0，有品牌溢价>1.0）
  × 增速系数 X.XX（营收/ EPS 增速 vs 30% 阈值，超阈值加成）
  × 毛利系数 X.XX（毛利趋势改善加成，恶化下调）
  × 财务健康系数 X.XX（无财务风险=1.0，每增加一个严重风险标签下调0.03-0.05）

基准PE = XX × X.X × X.XX × X.XX × X.XX = XX.Xx
基准净利（N+2E）= EPS XX × XX 亿股 = XX 亿
基准理论市值 = XX × XX.X = XXX 亿
基准弹性 = (XXX - 现市值) / 现市值 = +XX%
```

**财务健康系数说明**：

| 财务风险标签数 | 财务健康系数 | 说明 |
|:---:|:---:|:---|
| 0 | 1.00 | 财务健康 |
| 1-2个🟡 | 0.95 | 轻微风险，估值略打折扣 |
| 3-4个🟡或1个🔴 | 0.90 | 中等风险，估值折扣明显 |
| ≥5个🟡或≥2个🔴 | 0.85 | 严重风险，估值大幅折扣 |

四个系数的取值逻辑必须写清，不能凭空给数。

### 第 8 步：催化估值双轨 + 概率加权

把第 3 步的催化落到市值增量上。

| 催化 | 计算过程 | 结果 | 来源信源 |
|------|---------|------|---------|

每行催化：说明它调整了哪个系数 / 哪档 PE，算出催化市值，减基准得增量，再乘执行折扣/兑现概率。

汇总：催化合计 → 概率加权（每条催化给置信度）→ 综合估值 → 综合弹性。

### 第 9 步：风险情景量化（四档 + 财务健康影响）

| 情景 | N+1E EPS | PE 倍数 | 理论市值 | 弹性 | 触发条件 |
|------|---------|---------|---------|------|---------|

四档：乐观 / 中性（基准）/ 谨慎 / 悲观。弹性 = (理论市值 - 现市值) / 现市值。

**悲观情景必须考虑财务健康风险**：
- 若资产负债率>70%，悲观情景中可增加「融资受限导致扩产延迟」分支
- 若经营现金流/净利润<0.7，悲观情景中可增加「利润虚增、估值下修」分支
- 若利息覆盖<3，悲观情景中可增加「利率上行侵蚀利润」分支

底部 blockquote 给"消极过滤器"结论：有无亏损记录、现金流正负、营收净利趋势、研发投入稳定性——负面过滤是否通过。

### 第 10 步：仓位纪律 + 验证时间线

**仓位表**：

| 维度 | 建议 |
|------|------|

## 质量底线

1. **信源可溯**：每个关键数字/判断都能追到 T1-T3 信源
2. **计算可验**：中枢加权、催化增量、弹性百分比、财务健康系数，计算链完整可复算
3. **财务健康必评**：v2.5起，任何完整估值落地都必须包含财务健康评估，不允许跳过
4. **纪律闭环**：有目标价、有止损、有验证时间线、有条件分支
5. **诚实标注**：单信源标单信源，基数失真标失真，待测标待测
6. **不预测行情**：只做估值落地和仓位纪律，不做短线方向预测
7. **财务风险影响估值**：财务风险标签必须在第7步中枢加权、第9步风险情景、第10步仓位纪律中被引用
