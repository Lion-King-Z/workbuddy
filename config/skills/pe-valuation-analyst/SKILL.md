# pe-valuation-analyst v2.4

> **版本说明**：v2.4 以 WorkBuddy 共享研报「杰华特 (688141)」和「和林微纳 (688661)」的**PE双轨估值分析章节**为最高权威模板，一比一复刻其「估值摘要表」「盈利预测表」「理论PE推导」「安全边际评估」「可比公司估值参考」「PE打低路径」「基准仓位建议」的字段与格式。原有星大估值六条铁律、三大PE陷阱模板、行业增速容错率、基准估值、催化增量、风险情景量化、仓位纪律等核心方法论**全部保留**，但输出格式必须严格对齐下方模板。

## 一、核心承诺

给定一只标的和它的产业链故事，跑完 PE 估值落地工作流，返回一份清晰的、信源可溯源的估值报告：

`行业增速容错率 → PE陷阱检测 → 决策摘要 → 估值消化路径 → 催化状态 → 竞争格局 → 多档PE/PEG校验 → 基准估值 → 消息催化增量 → 风险情景量化 → 仓位纪律 + 验证时间线`

最终输出必须能直接嵌入「投资审判团【WorkBuddy】」HTML 研报的**维度三：PE双轨估值分析**章节。

## 二、核心哲学（保留 v2.3）

> **铁律一**：PE = 市场对增长的定价。PE高低不直接决定安全边际，增速持续性才决定。
> **铁律二**：价值判断是动态的，估值必须每季度重估。
> **铁律三**：周期行业看增速拐点，不看PE。
> **铁律四**：高增速行业给容错率，选对行业比选对个股更重要。
> **铁律五**：静态PE/TTM/预期PE三种本质不同，不能混淆。
> **铁律六**：合理估值是相对值/性价比，B2B品牌系数取1.0，毛利是壁垒代理变量。
> **铁律七**：重视增速持续性的确认，但「确认后加仓」是稳健建议，不是一票否决。

### 三大PE陷阱模板（保留）

| 模板 | 核心症状 | 关键验证点 |
|:---|:---|:---|
| **泡泡玛特型** | 近1-2季为历史营收峰值，PE看起来低 | 下一季报能否维持峰值季度70-80%营收？ |
| **美团型** | 增长引擎单一脆弱 | 是否有多个独立增长引擎？ |
| **铜矿型** | 商品价格高位→利润暴增→PE极低 | 商品价格还能涨吗？ |

### PE陷阱快速检测卡

```
□ 最近2季是否为营收历史峰值？ → 是→⚠️高基数风险
□ 增长引擎是否单一且脆弱？ → 是→⚠️增长脆弱风险
□ 利润是否与商品价格强相关？ → 是→⚠️周期位置风险
□ 近8季营收增速是否连续2季减速？ → 是→⚠️增速拐点风险
```

**陷阱仓位建议**：任何风险亮灯 → 建议仓位降至正常仓位的60-70%，但需结合B×G赔率综合判断。

## 三、工作流 — PE 估值落地完整流程

### 第 0 步：取数与信源分级

落地前必须取齐下列数据并标注来源：

| 数据项 | 首选源 | 信源等级 |
|--------|--------|---------|
| 现价/涨跌/市值/股本 | tdx-connector 行情 | T2 |
| 静态/TTM EPS、最新财报 | tdx-connector 财务 | T2 |
| 多档一致预期 EPS | tdx-connector 一致预期 | T3 |
| 行业增速 | tdx-connector / 财联社行业分析 | T2/T3 |
| 产业链卡位/订单/客户结构 | ima-mcp 原文库 → 财联社 | T1 |
| 研报深度推荐/行业格局 | IMA原文可查看库 → 财联社/证券时报 → 公开研报 | T1/T3 |
| 公告/重大事项 | 公司公告/交易所公告 | T0 |
| 历史PE分位图 | tdx-connector K线 | T2 |
| 同业可比公司PE/增速/毛利 | tdx-connector / 东方财富/同花顺 | T2/T3 |

### 第 0.5 步：行业增速容错率

在算个股之前，先算行业：

```
行业：{细分行业名称}
近3年行业营收 CAGR：XX%（来源：{T2/T3}）
当前行业阶段：高速成长期 / 稳定增长期 / 成熟期 / 衰退期
行业增速容错率：高(>20%) / 中(10-20%) / 低(<10%)
```

**判定**：
- 高容错率（CAGR >20%）：可适当放宽个股估值要求
- 中容错率（10-20%）：必须精选个股
- 低容错率（<10%）：个股选择零容错

### 第 0.6 步：PE 陷阱检测

按「三大PE陷阱模板」+ 四道关卡检测：

| 关卡 | 检测项 | 判定标准 | 结果 |
|--------|---------|------|------|
| 低PE陷阱 | 静态PE、近8季营收/净利趋势、最新季报同比 | PE低+增速连续2季减速 → 🔴 | — |
| 高基数效应 | 去年同期单季营收、是否为历史峰值 | 去年同期峰值+维持难度高 → ⚠️ | — |
| 周期顶部PE | 行业周期位置、商品价格、K线 | 周期中期偏上 → ⚠️ | — |
| 增长脆弱性 | 盈利能力波动、增长引擎数量、竞争格局 | 来源单一+盈利脆弱 → ⚠️ | — |

## 四、研报输出模板 — 维度三：PE双轨估值分析

> **强制要求**：当本 Skill 作为独立输出或嵌入「投资审判团【WorkBuddy】」时，PE估值分析章节必须严格按以下模板输出。字段名、表格列数、顺序不得改动。

```markdown
👤 PE分析师

**信号：{买入/增持/观望/卖出/强烈卖出}** ｜ **基准{市值}亿({股价}元) = 当前透支{+/-XX.X%}** ｜ **催化{市值}亿({股价}元) = 空间{+/-XX.X%}**

**概率加权综合：{市值}亿 / {股价}元 → {+/-XX.X%}**
```

### 4.2 PE陷阱检测

```markdown
### PE陷阱检测：{结论}

- ✅ {证据1}
- ✅ {证据2}
- ⚠️ {风险点}（若存在）
```

**规范**：
- 必须先说结论
- 至少2-3条证据/风险点
- 若陷阱亮灯，必须在后续估值中给出折价处理

### 4.3 双轨估值表（估值摘要表）

必须输出表格，列名固定为：

| 情景 | 方法 | 目标市值 | 目标股价 | 弹性 |
|:---|:---|---:|---:|---:|
| 基准(保守) | {PS 12x × 2026E营收 XX亿} | {XXX}亿 | {XX.X}元 | {-XX.X%} |
| 加权中枢 | {中枢PS X.Xx × 折价+催化概率调整} | {XXX}亿 | {XX.X}元 | {+XX.X%} |
| 催化(乐观) | {PS Xx × 2027E营收 XX亿折现+...} | {XXX}亿 | {XX.X}元 | {+XX.X%} |

**表格规范**：
- 列名必须精确为「情景 / 方法 / 目标市值 / 目标股价 / 弹性」
- 情景固定为：基准(保守) / 加权中枢 / 催化(乐观)
- 方法必须说明具体倍数和基数（如 PS 12x × 2026E营收 44.5亿）
- 市值单位「亿」，股价单位「元」，弹性用百分比
- 亏损或无法计算时写「—」或「N/A」

### 4.4 可比公司估值参考表

必须输出表格，列名固定为：

| 公司 | PS(倍) | 增速 | 毛利率 |
|:---|:---:|:---:|:---:|
| {公司A} | {XX-XXx} | {描述} | {XX%} |
| {公司B} | {XX-XXx} | {描述} | {XX%} |
| {公司C} | {XX-XXx} | {描述} | {XX%} |
| {标的本身} | {~XXx} | {+XX%} | {XX%} |

**规范**：
- 至少3家可比公司 + 标的自身
- 列名必须精确为「公司 / PS(倍) / 增速 / 毛利率」
- 底部给出中枢PE调整公式：
  ```
  中枢PS调整 = 可比15x × 品牌{X.XX} × 增速{X.XX} × 毛利{X.XX} = {XX.X}x
  ```

### 4.5 理论PE推导

```markdown
### 理论PE推导

可比均值 **{XX.XX}×**（{可比公司} {年份}E PE均值）
× 龙头品牌溢价 **{X.XX}**（{溢价依据}）
× 增速调整 **{X.XX}**（{净利润增速>XX%}）
× 毛利率调整 **{X.XX}**（{结构件/业务拖累整体毛利率}）
= **{XX.XX}×** 理论PE
```

**规范**：
- 必须展示四步乘法链
- 每个系数必须有依据说明
- 结果加粗

### 4.6 安全边际评估

```markdown
### 安全边际评估

❗ {严重高估/高估/合理/低估/严重低估}

当前价 **{XX}元** vs 基准估值 **{XX.X}元** → 溢价 **{+XXX%}**
当前价 **{XX}元** vs 催化估值 **{XX.X}元** → 溢价 **{+XXX%}**

**{N}重估值交叉验证（PE/PB/PS）一致指向{结论}。**
```

**规范**：
- 必须给出当前价 vs 基准估值、当前价 vs 催化估值两组对比
- 溢价率必须计算
- 必须给出定性评估（严重高估/高估/合理/低估/严重低估）
- 必须说明交叉验证结论

### 4.7 PE打低路径

```markdown
### PE打低路径

{静态PE} → {N+1E} → {N+2E} → {N+3E}。{N+3}年净利润 {XX}亿，若给予 {XX}-{XX} 倍PE，对应市值 {XXX}-{XXX}亿({XX}-{XX}元)。市场定价反映{催化成功概率约XX%}。
```

**规范**：
- 必须给出从静态PE到预期PE的逐年路径
- 必须给出远期净利润和对应市值区间
- 必须给出市场对催化成功概率的隐含定价

### 4.8 基准仓位建议

```markdown
### 基准仓位建议

受景气度弹性上限 {X%} 约束 → **{X%}**。{但当前股价已高于基准估值+安全边际，建议控制仓位。/估值处于合理区间，可按上限配置。}
```

**规范**：
- 必须引用景气度弹性上限
- 给出基准仓位数字
- 说明是否需要控制仓位

### 强制规则

- T0 与 T1-1/T1-2/T1-4/T1-5 为**结论级信源**，可直接用于投资结论。
- T1-3（快讯）为**数据级信源**，重要数据需 T1-2/T2 交叉验证。
- T2 为**数据级信源**，可直接用于行情、财务、估值数据。
- T3 为**参考级信源**，需与 T0/T1/T2 交叉验证后使用。
- T4 为**线索级信源**，**禁止写入结论**，必须反向验证升级为 T0-T2 后方可使用。
- 所有关键判断必须 `✅双重验证`（≥2个独立信源，且至少一个 T0/T1/T2）。

### 交叉验证规则

| 结论类型 | 最低信源要求 |
|:---|:---|
| 直接写入结论 | ≥1个 T0/T1-1/T1-2；或 ≥2个独立 T1-3/T2/T3 |
| 紧缺/涨价/订单判断 | ≥1个 T1 + ≥1个 T2（独立来源） |
| 定量数据引用 | 至少 T2 及以上，且数据在有效期内 |
| 估值参数（PE/PS/增速） | ≥2个独立 T2/T3，或 1个 T2 + 1个 T3 |

**独立性判定**：同一媒体的多个报道只算 1 个独立信源；同一事件的公司公告 + 财联社报道 = 2 个独立来源。
