# 每日窄门筛股 Skill v2.0

**定位**：从第一性原理出发——每条信号都有"正确概率"，工作不是过滤掉低概率的信号，而是给每条信号分配概率权重，用仓位控制风险。

---

## 第一性原理

### 投资本质不是"筛选"，是"定价"

市场给了每个标的当前价格——这个问题是：**市场定价与真实价值的差距有多大？** 差距 = 预期差。预期差越大，机会越大。

把这个逻辑迁移到信号筛选上：
- 瓶颈弱（B=2）的信号不一定没价值——如果预期差极大（市场完全没定价），2分瓶颈也可能比5分瓶颈赚得多
- 新信号（S=1）不一定是噪音——如果信息刚出现、市场还没反应，这是信息优势，不是噪音
- 单一来源不一定是谎言——如果来源是"原文可查看[浑水调研]"的一手纪要，比三个二手新闻加起来都可靠

### 四个维度的独立性

每个维度衡量不同的东西，不可互相替代：

| 维度 | 衡量什么 | 高意味着 | 低意味着 | 权重 |
|:---|:---|:---|:---|:---:|
| **瓶颈强度 B** | 供给刚性的"物理事实" | 稀缺会持续很久 | 即便稀缺也是暂时的 | 25% |
| **证据累积度 E** | 从首次出现到现在，累积了多少新信息 | 故事越来越可信 | 反复出现但无增量 | 25% |
| **确定性 C** | 信源质量 + 关键假设被验证的比例 | 判断大概率是对的 | 有很多待验证的假设 | 25% |
| **预期差 G** | 市场定价 vs 真实紧缺的背离 | 市场还没发现这个机会 | 市场已经充分定价 | 25% |

**赔率公式**：高B×高G = 强供给约束 + 市场没发现 = 最大机会（典型案例：早期的六氟化钨，B=5, G=5）
**胜率公式**：高E×高C = 证据厚 + 信源可靠 = 大概率判断正确（典型案例：现在的电子布，E=7, C=7）

### 仓位 = 赔率 × 胜率

不是"B≥3且S≥3才给操作建议"，而是：

`综合分 = B×25% + E×25% + C×25% + G×25%`

不同得分区间对应不同仓位，而非不同"操作标签"：

| 综合分 | 仓位建议 | 思维框架 |
|:---:|:---|:---|
| ≥8.0 | 重仓/核心持仓（仓位上限×100%） | 高赔率+高胜率共振 |
| 6.5-7.9 | 标准仓位（仓位上限×70%） | 赔率或胜率有一端略逊 |
| 5.0-6.4 | 试探性仓位（仓位上限×30%） | 有逻辑但证据不足，小注押 |
| 3.5-4.9 | 观察清单（不持仓） | 值得跟踪，等关键验证点触发 |
| <3.5 | 放弃追踪 | 不浪费注意力，移除信号池 |

**原则**：低确定性高赔率（B=5/C=3/G=5）> 高确定性低赔率（B=3/C=5/G=2）。前者仓位轻但上限高，后者仓位重但上限低——这是专业投资者的核心判断。

## 工作流：四维加权评分

### Step 0：信号汇总与去重

```
1. 读取今日三条产出线的全部信号
2. 按行业/环节去重合并
3. 标注来源：紧缺度日报 | 电话会 | 推送 | 聚宝盆缓存
4. 形成候选信号池
```

🔴 **CHECKPOINT — Step 0 出口**：确认三条产出线至少一条有数据。若三条均为空→用昨日窄门筛股结果+浑水跟踪表做延续性分析，标注「今日无新产出」。信号数<3条→正常输出，不凑数。

### Step 1：瓶颈强度 B 评分（0-10分）

从产业链瓶颈投研skill的方法论提取，5个维度，每维 0-2 分：

| 维度 | 0分 | 1分 | 2分 |
|:---|:---|:---|:---|
| **供给刚性** | 扩产<12个月 | 扩产12-18个月 | 扩产>18个月，有认证/建厂硬约束 |
| **量化缺口** | 无数字，纯定性描述 | 有数字但未交叉验证 | 有量化缺口且被多源确认 |
| **不可替代性** | 替代品充足且价格正常 | 替代品存在但也在涨价 | 无替代品，或替代品被同一瓶颈卡住 |
| **价格传导** | 未涨价或涨价被下游拒绝 | 涨价已被下游部分接受 | 涨价全链条传导，下游确认接受 |
| **需求端刚性** | 需求可推迟/可替代 | 需求有粘性但短期可调节 | 需求无弹性（停产=断供下游） |

**B计算方法**：5维度加总 = B（0-10分）

**降权而非过滤**：B=2的信号不丢弃，只是综合分大打折扣。如果其他三维（E/C/G）极高，仍可能进入观察清单。B越低，需要的E×C×G补偿越高。

🔴 **CHECKPOINT — Step 1 出口**：B≥8或B≤2 → 标注极端值。B≤2且E+C+G<12 → 综合分≤3.5可能性高，标记「大概率进退出追踪」。

### Step 2：证据累积度 E 评分（0-10分）

**核心创新——不是数天数，而是数"信息增量"**。

每次信号被提及时，判断这次提及带来了多少**新信息**：

| 信息增量类型 | 加分 | 示例 |
|:---|:---:|:---|
| 首次出现（新信号） | +2.0 | 从0到1，建立信息锚点 |
| 新增量化数据（之前没有的具体数字） | +2.0 | "缺口约500吨"是第一次出现的数据 |
| 新增独立信源（第二个来源确认） | +2.0 | 电话会+产业数据双源确认 |
| 新增供给事件（停产/断供/扩产延迟） | +1.5 | "日本7/1宣布停产"是新事件 |
| 新增下游验证（下游确认接受涨价） | +1.5 | "客户已接受Q3涨价"是新环节确认 |
| 数据边际恶化（缺口扩大/涨幅度增加） | +1.0 | 缺口从30%→50%，幅度从涨20%→涨40% |
| 重复提及但无新信息 | +0 | "仍然紧缺"——不加分，信息冗余 |
| 趋势逆转（增强→减弱） | -1.0 | 之前涨价扩大，现在涨幅收窄 |
| 新替代方案出现 | -0.5 | 有替代路径被验证可行 |

**E从0起算，上限10分。** 每次有新信息才加分，没有新信息=不加分。这样"真正在发酵"的信号自然累积高分，"反复炒冷饭"的信号自然停滞低分。

**E的特例**：当信号有供给断崖级别的事件（如"日本7/1停产"），E直接给初始分 +3.0，因为一个硬事件的信息量远超N天重复。

🔴 **CHECKPOINT — Step 2 出口**：E≤2且非供给断崖事件 → 证据极弱，综合分上限被拉低，标记「证据不足，仓位建议不超试探级」。

### Step 3：确定性 C 评分（0-10分）

衡量"这个判断有多大可能是对的"，两维度加权：

**C1 — 信源质量（0-5分）**：

| 信源 | C1 |
|:---|:---:|
| 浑水调研原文（P0）+ 法定公告（A级）交叉验证 | 5 |
| 浑水调研原文（P0）+ 行业数据（B级）交叉验证 | 4 |
| 浑水调研原文（P0）单一来源 | 3 |
| WebSearch公开数据 + 行业数据 | 2 |
| 单一WebSearch新闻 | 1 |

**C2 — 关键假设验证率（0-5分）**：

信号背后有多少关键假设已被验证：

| 验证率 | C2 | 含义 |
|:---:|:---:|:---|
| ≥80%（几乎所有假设都被验证） | 5 | 高度确定 |
| 60-80% | 4 | 主要假设已验证 |
| 40-60% | 3 | 部分验证，有不确定性 |
| 20-40% | 2 | 多数假设待验证 |
| <20% | 1 | 基本全在假设阶段 |

**C = (C1 + C2) / 2，四舍五入到0.5**

**不确定不是禁忌**：C=3不等于"不推荐"——如果B=5/G=5（强瓶颈+大预期差），C=3只是意味着"先试探性仓位，等C提升到4以后再加重"。低C是高赔率早期的必然代价。

🔴 **CHECKPOINT — Step 3 出口**：C≤2 → 确定性极低，止损收紧（C/10→止损比例加倍收紧），仓位上限强制×30%。若B×G极高（≥40）可豁免但标注「高赔率低确定性，止损必紧」。

### Step 4：预期差 G 评分（0-10分）

衡量"市场定价 vs 真实紧缺的背离程度"。

| 维度 | 0-2分（低预期差） | 3-5分（中等预期差） | 6-8分（高预期差） | 9-10分（极端预期差） |
|:---|:---|:---|:---|:---|
| **市场关注度** | 已成为热门话题，街头巷尾都在聊 | 被少数机构关注但未扩散 | 仅浑水调研/专业渠道提及 | 完全没人在讨论 |
| **估值水位** | PEG>2.0，PE远超行业中枢 | PEG 1.5-2.0 | PEG 1.0-1.5 | PEG<1.0，估值被忽视 |
| **催化剂时点** | 催化剂已兑现，利好出尽 | 1个月内催化剂 | 1-3个月催化剂 | 3个月以上远期催化剂 |
| **持仓拥挤度** | 已被大量机构重仓 | 开始有机构建仓 | 仅少数先行者持仓 | 几乎无机构覆盖 |

**G = 四个子维度得分加总（上限10分）**

**G子维度量化规则**（每维0-2.5分，避免纯定性判断）：

| 子维度 | 0-0.5分 | 1-1.5分 | 2-2.5分 |
|:---|:---|:---|:---|
| **市场关注度** | 微博热搜/各大财经头条在推，WebSearch搜相关新闻>20条 | 仅浑水调研+少数机构提及，WebSearch搜相关新闻5-20条 | 几乎无公开讨论，WebSearch搜相关新闻<5条，仅专业渠道 |
| **估值水位** | PEG>2.0或PE高于行业中枢50%+ | PEG 1.5-2.0或PE高于行业中枢20-50% | PEG<1.0或PE低于行业中枢，或无人覆盖无PE数据 |
| **催化剂时点** | 催化剂已兑现超过1个月 | 1个月内催化剂即将兑现 | 3个月以上远期催化剂，市场未开始定价 |
| **持仓拥挤度** | 已被多家头部机构重仓，券商覆盖≥5家 | 开始有1-2家机构覆盖，持仓分散 | 几乎无机构覆盖或持仓，券商覆盖0-1家 |

**G判据获取路径**：
- 市场关注度 → WebSearch搜索量 + 雪球热帖数
- 估值水位 → tdx-connector PE数据 + 行业中枢PE
- 催化剂时点 → 浑水跟踪表"供给事件"列的时间节点
- 持仓拥挤度 → 券商覆盖家数（WebSearch"标的简称 + 研报"）

🔴 **CHECKPOINT — Step 4 出口**：G≥8 → 极高预期差，但需配合E验证——如果E≤3（证据不足），预期差可能是"真的没发生"而非"市场没发现"。标注「高G低E，需区分"未定价"vs"未发生"」。

**预期差与时间的关系**：高预期差通常伴随低确定性（C低），这是合理的——市场没发现的东西你也更难验证。随着时间推移，确定性上升（C↑），但预期差会缩小（G↓）——当C到顶、G到底时，就是退出的时机。

### Step 5：仓位建议

**仓位上限** = 景气度skill 对应阶段的上限（如3级=10%）

**仓位系数** = 综合分映射：

| 综合分 | 仓位系数 | 实际仓位 |
|:---:|:---:|:---|
| ≥8.0 | 100% | 仓位上限×100% |
| 6.5-7.9 | 70% | 仓位上限×70% |
| 5.0-6.4 | 30% | 仓位上限×30% |
| 3.5-4.9 | 0%（观察） | 不持仓，跟踪关键验证点 |
| <3.5 | — | 放弃追踪 |

**止损线**：
```
止损比例 = 标准止损比例(15-20%) × (1 - C/10)
```
C越低（确定性越低），止损越紧——这是用仓位纪律控制不确定性。

**目标价**：
```
目标价 = 一致预期EPS(N+1) × 行业中枢PE × (1 + B溢价 + G溢价)
B溢价 = (B - 5) × 3%（B=10时溢价15%，B=5时溢价0%，B=2时折价-9%）
G溢价 = (G - 5) × 2%（G=10时溢价10%，G=5时溢价0%，G=2时折价-6%）
```
B和G独立贡献溢价——瓶颈强+预期差大=双重溢价，瓶颈弱+无预期差=折价。

🔴 **CHECKPOINT — Step 5 出口**：目标价 < 当前价×0.8 → 标注「目标价远低于现价，基本面不支持当前估值」。仓位建议 > 15% → 标注「高仓位建议，确认无黑名单条件」。

🛑 **STOP — Step 5 出口**：若当前持仓某标的的综合分 < 3.5（退出追踪），必须输出「建议平仓」而非「持有」或「观察」。这是红线。

## 核心原则

| # | 原则 | 含义 |
|:---|:---|:---|
| 1 | **概率优先，不二元过滤** | 每个信号都有权重，权重低≠噪音，只是仓位轻 |
| 2 | **赔率×胜率，不堆叠门槛** | 低确定性高赔率可以比高确定性低赔率更有价值 |
| 3 | **证据累积，不天数计数** | 同一信号被提10次但无新信息=E不加分；提3次但每次有新数据=E狂加分 |
| 4 | **预期差驱动，不信号驱动** | 市场已经知道的稀缺=不是值得交易的稀缺 |
| 5 | **仓位控制风险，不过滤控制风险** | 低确定性标的给1%仓位比过滤掉更好——你保留了对这个方向的注意力 |
| 6 | **评分独立加权，不级联门槛** | B低但G高 → 仍可能是好机会，只是仓位要轻 |

## 风险降级规则（替代绝对禁止）

不设"禁止做X"，而是"如果缺少Y，则评分降权"：

| 规则 | 降权 |
|:---|:---|
| 目标价无计算依据 | G降1分（无法判断预期差） |
| 无历史追踪数据（首次出现） | C降1分（确定性不足） |
| 无量化数据支撑 | B降1分（瓶颈认定弱化） |
| 所有标的评分均匀分布在中段 | 标注"评分趋同，批次信息不足，本周可能无强信号" |
| 持仓标的需要调用三分析师辩论做深度复核 | 但日常筛股只给轻量建议 |

## 反例黑名单（dim9 — 绝对不要做的行为）

以下行为会直接破坏评分体系的概率性质，必须避免：

| # | 反模式 | 为什么不能做 | 正确做法 |
|:---|:---|:---|:---|
| 1 | **对B≤3的信号直接丢弃** | 概率权重≠二元过滤；低瓶颈+高预期差可能是好机会 | B≤3时仓位系数自动打折，但不从信号池移除 |
| 2 | **对S=1（首次出现）的信号判死刑** | 首次出现=信息优势，不是噪音。只要B×G≥20即可进观察清单 | 综合分<3.5才放弃，不单独因S=1放弃 |
| 3 | **对持仓标的综合分<3.5仍给"持有"** | 🛑红线。跌破退出阈值却建议持有=对抗评分体系 | 输出"建议平仓"，附平仓理由 |
| 4 | **不读浑水跟踪表直接评E分** | E维度依赖历史数据提取信息增量，不读历史=E分是猜的 | 先Read浑水跟踪_YYYY-MM.md，逐条对比历史提及记录 |
| 5 | **用天数替代证据累积评E** | 连续出现10天但无新信息=E=2（只有初始分）；出现3天但新供给事件+新数据=E=5.5 | 必须逐条检查每次提及的"信息增量"，不数天数 |
| 6 | **跳过多源验证直接定C** | C1信源质量需要判断来源层级，单一WebSearch和浑水调研原文差2分 | Step 0已标注来源→Step 3直接对号入座 |
| 7 | **评分趋同时硬凑操作建议** | 全市场无强信号是正常现象，硬推买入=制造FOMO | 标注"无强信号日"，保留持仓监控+退出追踪即可 |
| 8 | **把目标价当精确数字而非区间** | EPS(N+1)×中枢PE是两个估计值的乘积，误差至少±15% | 目标价输出为区间（当前价→目标价），而非单点 |
| 9 | **对已持仓标的和对未持仓标的用同一标准** | 已持仓有沉没成本和持仓纪律约束，综合分跌破3.5是强制平仓信号 | 持仓标的加STOP规则，未持仓标的只是从观察清单移除 |
| 10 | **省掉"变化"列的昨日对比** | 没有昨日对比=看不到趋势方向=失去动态追踪价值 | 核心持仓监控表必须标注每个维度的增减方向 |

## 失败模式与异常处理（三段式）

| 触发条件 | 一线修复 | 仍失败兜底 |
|:---|:---|:---|
| 今日三条产出线均未执行 | 检查聚宝盆是否有今日产出标题 | 用昨日窄门筛股结果+浑水跟踪表做延续性分析，标注「今日无新产出」 |
| 浑水跟踪_YYYY-MM.md 文件不存在 | 改为读取紧缺度追踪总表提取历史数据 | E维度从0起算（无历史累积），标注「历史数据缺失」 |
| 持仓标的PE(TTM)/EPS数据无法获取 | tdx-connector → WebSearch搜「标的 2026年 预期EPS」→ 雪球F10 | C降1分，仓位建议降一档，目标价标注「PE数据缺失」 |
| search_knowledge API 频控 (code=110021) | 等待30秒重试1次 | 跳过聚宝盆搜索，直接从本地文件读取今日产出 |
| 聚宝盆缓存查询超时 | 放弃缓存查询 | 全部从本地文件读取，标注「缓存不可用」 |
| 信号数量过少（<5条） | 正常输出，不凑数 | 标注「低信号日」，不硬推操作建议 |
| 所有信号集中在3-5分区间 | 检查是否存在评分趋同（同一来源→C相同；同一缺口大小→B相同） | 标注「无强信号日，批次信息不足」，降低输出置信度 |
| 同一环节多个标的（如电子布→巨石/宏和/国际复材） | 只推综合分最高的1-2个 | 标注「同环节多标的只列Top2」 |
| 持仓某标的综合分跌破3.5 | 输出「建议平仓」，给出平仓理由 | 🛑 STOP——不输出「持有」或「观察」 |

🛑 **STOP规则**：持仓标的综合分<3.5→必须输出平仓建议。这是唯一硬性红线。
