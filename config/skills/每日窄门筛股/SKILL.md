# 每日窄门筛股 Skill v4.9

**定位**：买入信号**总调度** + **首席裁决官**。不自己硬算——专业的事情交给专业的Skill做；最终由窄门筛股综合各专业输出，给出排序与审判。

**v4.9 核心升级**（2026-07-18）：
1. 🆕 **子Skill覆盖完整性检查** — 生成报告前强制确认四个子Skill全部返回，关键字段缺失标红
2. 🆕 **数据一致性校验** — 跨Skill比对同一标的的关键数据（营收增速/净利增速），差异>10%标红
3. 🆕 **风险聚合矩阵** — 汇总产业链瓶颈、景气度、PE、个股风险扫描器四方风险标签，统一呈现
4. 🆕 **评分敏感度分析** — 对Top5展示B/G/C权重±5%变化时排名是否改变
5. 🆕 **子Skill分歧预警** — 当B/G/C任两个差距>3分时，强制标注「维度分歧，降低置信度」
6. 🆕 **执行摘要（Executive Summary）** — 报告顶部100字内说清今日最强方向、Top3、最大风险
7. 🆕 **风险调整评级** — 综合分相同但风险标签更多者降级，让风险直接参与排序
8. 🆕 **估值方法适配性检查** — 对0-2级早期标的用TTM PE主导时标红警示
9. 🆕 **数据来源索引** — 所有关键数据集中列出：数据项/来源/等级/日期
10. 🆕 **方法论说明** — 报告末尾固定呈现B/G/C权重、C乘数映射、黄金赔率规则

## 架构：七层委派（B/阶段/G+PE/风险/行情/事件映射/完整性校验）

```
信号池（窄门筛股收集）
    │
    ├─→ 实时行情采集 ──────────→ 最新股价 + 近N日涨幅
    ├─→ 事件→产业链映射 ──────→ 大事件受益标的自动入池
    │
    ├─→ 产业链瓶颈投研skill  ──→ B(瓶颈强度) + 供给刚性类型 + 紧缺信号
    ├─→ 景气度skill          ──→ 产业阶段(0-6级) + 材料子生命周期
    ├─→ pe-valuation-analyst ──→ G(预期差) + PE估值 + 三层目标价 + 财务健康
    └─→ 个股风险扫描器       ──→ 14维风险画像 + 两融风控
    │
    ↓
窄门筛股汇总：完整性检查 + 数据一致性 → 涨幅风险检测 + 前瞻PE调整 → 三因子评分 + 敏感度分析 → 排序 → 风险调整评级 → 买入建议
    │
    ↓
v4.9 综合审判：宏观 + 产业 + B/G/C + PE + 财务健康 + 涨幅 + 风险聚合 + 催化剂日历 八维裁决
```

窄门筛股不重复造轮子——每个维度的评分由最懂它的Skill出；最终审判由窄门筛股综合输出。

## 工作流

### Step 0：信号收集（窄门筛股自己做）

1. 从三来源提取所有被提及的品种/标的
2. 去重合并，形成候选池
3. 标注来源和初步紧缺信号强度
4. 🔴 当日无产出→用昨日结果延续，标注"无新增信号"

### Step 0.5：实时行情采集（窄门筛股自己做）

对候选池中每个有股票代码的标的，通过 **tdx-connector（通达信MCP）** 获取实时行情：

```
tdx_quotes(code=股票代码)
→ 提取：最新价 / 涨跌幅 / 近5日涨幅 / 近20日涨幅 / 近60日涨幅
```

若tdx-connector不可用，降级为 **akshare-stock** 或 **westock-data**。

**产出**：每个标的的 `当前价` + `近5日涨跌幅` + `近20日涨跌幅` + `近60日涨跌幅`

**⚠️ 不做全市场科创拥挤度扫描**：该指标Token消耗高（约3-6K/次），且对选股决策为弱辅助信号，v4.3 起不在每日窄门筛股中采集。情绪温度通过「综合分+评级+风险提示」间接体现。

### Step 0.6：事件驱动产业链前瞻映射（窄门筛股自己做）

**核心逻辑**：重大事件（如美光财报超预期）→ 产业链上下游受益标的 → 即使这些标的尚未出现在今日推送中，也应纳入候选池。

**映射规则**：

| 事件类型 | 受益方向 | 示例标的 |
|:---|:---|:---|
| 存储原厂财报超预期 | 封测/材料/设备 | 长电科技/雅克科技/中船特气 |
| 英伟达GPU放量 | 光模块/PCB/散热 | 新易盛/沪电股份/高澜股份 |
| AI资本开支上调 | 服务器/柴发/液冷 | 工业富联/潍柴重机/英维克 |
| 晶圆厂扩产 | 硅片/特气/靶材 | 有研硅/中船特气/江丰电子 |
| 封装技术突破(CoWoS等) | 封装基板/ABF/玻璃基板 | 深南电路/生益科技 |
| 终端产品发布(AI眼镜等) | 光学/显示/MCU | 水晶光电/韦尔股份 |

**操作**：
1. 扫描今日信息推送中的S级/A级信号
2. 对每个S/A级信号按上表推导受益方向
3. 将推导出的标的追加到候选池（标注来源为"事件映射"）
4. 事件映射标的也走完整委派评估流程

**🆕 反向验证规则**（v4.6）：

事件映射推导出的标的，在进入候选池之前必须通过以下验证：

| 验证步骤 | 要求 | 不满足时 |
|:---|:---|:---|
| ①信源交叉验证 | 该标的至少被1个T1-2级或2个T3级独立来源提及（即便推送未直接列名，电话会/紧缺度日报有提及也可以） | 标注「⚠️事件映射未验证」 |
| ②产业链位置验证 | 该标的确实处于传导链的受益位置（非攀附概念） | 剔除出候选池，另行标注「事件误映射」 |
| ③基本面门槛 | B≥2 或 近60日有公开业绩/订单信息 | 保留候选但标注「低基本面支撑」 |

经验证通过的标的标准「事件映射✅」；验证不通过但保留的标注「事件映射⚠️」+不通过原因。

### Step 0.7：排名动态对比（v4.6，窄门筛股自己做）

**核心逻辑**：每日窄门筛股不是孤立的——昨日Top10的变化方向本身就是信号。上升意味着趋势强化，下降/退出意味着信号衰减。

**操作**：

1. 读取昨日窄门筛股文件（`深度研究/<昨日周文件夹>/<昨日日期>_每日窄门筛股.md`），提取「一、买入建议排序」中的排名+标的+综合分
2. 若昨日文件不存在（如周一日无文件），标注「无可比数据（昨日无产出）」，跳过对比
3. 与今日排序结果对比：
   - **标注规则**：
     - 昨日未出现 → 🆕新进
     - 排名上升 ≥3位 → 🔥↑N
     - 排名上升 1-2位 → ▲↑N
     - 排名不变 → →持平
     - 排名下降 1-2位 → ▼↓N
     - 排名下降 ≥3位 → ⚠️↓N
     - 昨日Top10但今日不在Top10 → 💤退出Top10
   - **综合分变化**（补充参考）：上升≥0.5→走强 / 下降≥1.0→走弱

**融入输出**：在「一、买入建议排序」表格中增加「动态」列，在「二、🏆 黄金赔率」中标注黄金赔率标的的排名变化。

### Step 1：B 瓶颈强度 → 委派给「产业链瓶颈投研skill」

对每个候选品种，使用产业链瓶颈投研skill的以下模块：

**1a. 供给刚性分类**（Skill Step 1.5）
- 判定类型：产能约束型 / 认证锁定型 / 地缘切割型
- 分类决定后续催化剂路径

**1b. 四维度深度评分**（Skill Step 2）
- 不可替代性(30%) + 供应弹性(30%) + 集中度(20%) + 认知偏差(20%)
- 使用供给刚性类型对应的调整后权重
- 输出：加权总分 → 映射到B(0-10)

**1c. 紧缺六维信号检测**（Skill 第三章）
- 价格/产能/客户行为/库存/投资/跨市场验证
- 确认信号数 → 辅助B验证

**B赋值规则**：

| 四维加权总分 | B分数 |
|:---:|:---:|
| ≥4.0 (🔴Red alert) | 8-10 |
| 3.5-3.9 (🟠Orange) | 6-7 |
| 3.0-3.4 (🟡Yellow) | 4-5 |
| 2.0-2.9 | 2-3 |
| <2.0 | 0-1 |

⚠️ B≤2 → 标注「供给刚性不足」，排序靠后，不推荐但不隐藏。

### Step 2：产业阶段 → 委派给「景气度skill」

对每个候选品种，使用景气度skill：

**2a. 判定产业阶段**（Skill Step 1）
- 通用7级：0级(技术验证)→6级(价值陷阱)
- 半导体材料走子生命周期：①验证→②导入→③小批量→④国产替代

**2b. 阶段=赔率定位**
- 0.5级→3.5级 = 黄金投资区间（一年十倍摇篮）
- 3.5级之后 = 吃业绩兑现的钱，弹性降低
- 5级+ = 弹性释放，赔率下降

**阶段映射到催化剂链权重调整**：

| 阶段 | 对催化剂的含义 |
|:---|:---|
| 1-2级（导入期） | 近期催化剂密集，权重上调 |
| 3级（产能爬坡） | 中期催化剂关键 |
| 4级+（业绩释放） | 催化剂已大量兑现，远期格局为主 |

### Step 3：G 预期差 + PE估值 + 财务健康 → 委派给「pe-valuation-analyst」

对每个候选品种，使用pe-valuation-analyst：

**3a. PE陷阱检测**（Skill Step 0.6）
- 三大陷阱对号入座：泡泡玛特型/美团型/铜矿型
- 陷阱≠淘汰，陷阱=风险地图

**3b. 财务健康评估**（Skill Step 4.5，v2.5新增）
- 偿债能力：资产负债率/流动比率/利息覆盖/短债占比
- 运营效率：存货周转/应收周转/总资产周转
- 现金流质量：经营现金流/净利润/FCF/资本开支
- 输出：财务风险标签（如「利润含金量偏低」「短债占比高」）

**3c. 多档PE评估**（Skill Step 5）
- 静态PE / TTM PE / N+1E / N+2E
- PEG校验
- 增速持续性检验

**3d. 三层目标价**（Skill Step 6b/7）
- T1（认知启动价）= 基准净利 × 中枢PE × (1+B溢价)
- T2（紧缺确认价）= T1 × (1+近期催化溢价)
- T3（充分定价价）= T2 × (1+全催化溢价)
- 上行空间 = (T3-当前价)/当前价

**3e. G预期差评分**（从PE Skill的认知偏差+估值水位推算）

| G子维度 | 数据来源 |
|:---|:---|
| 市场关注度 | PE Skill的市场关注度判断 + WebSearch |
| 估值水位 | PE Skill的多档PE + PEG + 行业百分位 |
| 催化剂时点 | PE Skill Step 3催化剂追踪仪表盘 |
| 持仓拥挤度 | PE Skill的券商覆盖数 |
| 财务健康 | PE Skill Step 4.5财务风险标签 |

### Step 3f：涨幅风险检测（窄门筛股自己做）

基于 Step 0.5 获取的实时行情，对每个标的检测涨幅风险：

**检测维度**：

| 检测项 | 计算方式 | 风险信号 |
|:---|:---|:---|
| 短期冲高 | 近5日涨跌幅 | >20%→🔴高风险 / 10-20%→🟡注意 / <10%→🟢正常 |
| 中期累积 | 近20日涨跌幅 | >40%→🔴高风险 / 20-40%→🟡注意 / <20%→🟢正常 |
| 上行空间vs涨幅 | 预期上行空间 / 近20日涨幅 | <1.5x→涨幅已充分反映 / 1.5-3x→部分反映 / >3x→仍有空间 |
| 是否已反映预期 | 当前价 vs T1目标价 | 已超T1→预期已部分定价 / 逼近T2→接近充分定价 / <T1→预期未反映 |

**风险标注规则**：

| 条件 | 标注 |
|:---|:---|
| 近5日>20% 或 近20日>40% | ⚠️ 短期涨幅过大，追高风险 |
| 当前价已超T1 | ⚠️ 预期已部分反映，安全边际收窄 |
| 当前价逼近T2 | ⚠️ 预期接近充分定价，上行空间有限 |
| 上行空间/近20日涨幅 <1.5x | ⚠️ 涨幅已充分反映预期 |

### Step 3g：前瞻PE调整（窄门筛股自己做）

**核心逻辑**：PE Skill基于历史财报计算PE，但今日重大事件（如美光财报超预期）意味着未来盈利预期上修，TTM PE可能虚高。需要做前瞻性调整。

**调整规则**：

1. **事件→盈利预期修正**：
   - 若今日有产业龙头财报超预期（如美光），该链条上游标的的N+1E预期收入至少上调10-20%
   - 反映到前瞻PE = 当前市值 / (原N+1E × 1.1~1.2)

2. **涨价传导修正**：
   - 若产品已在涨价通道（如MLCC村田7/1提价），N+1E天然上修
   - 将涨价幅度折算进N+1E

3. **输出**：在PE评估中同时展示"TTM PE(历史)"和"前瞻PE(事件调整后)"，标注调整依据

### Step 3h：估值方法适配性检查（v4.9新增）

**核心逻辑**：PE Skill会输出多档PE，但窄门筛股需要判断「当前估值方法是否与产业阶段匹配」。

**检查规则**：

| 产业阶段 | 估值方法适配性 | 不匹配时标注 |
|:---|:---|:---|
| 0-2级（导入/验证期） | 应以N+2E/N+3E或PS为主，TTM PE参考权重低 | 若PE Skill以TTM PE作为首要判断依据→标红「早期标的不宜重看TTM PE」 |
| 3级（爬坡期） | TTM + N+1E并重 | 正常 |
| 4级+（放量/成熟期） | TTM PE + N+1E为主 | 正常 |

### Step 4：多维风险画像 → 委派给「个股风险扫描器」（v4.9升级）

**核心逻辑**：前5名标的必须调用个股风险扫描器，获取14维风险画像。风险画像不直接纳入综合分公式，但参与风险聚合与风险调整评级。

**调用方式**：对排序前5的每个标的，强制调用 @skill:"个股风险扫描器"（输入：股票代码 + 诊断日期）。

**输出用途**：
1. 在「四、风险聚合矩阵」中集中呈现来自风险扫描器的风险标签
2. 在「五、前5名两融价格区间对比」中完整呈现两融风控数据（保留原两融风控师能力）
3. 在买入建议表的风险提示中引用风险扫描器结论
4. 在极端情况下（强平区+踩踏烈度高）对买入建议评级做降级处理

**禁止行为**：
- 禁止窄门筛股自行用脚本或公式复刻风险扫描器的计算逻辑
- 禁止把风险扫描器直接纳入B/G/C综合分公式
- 禁止因单一风险标签将品种从排序中移除

### Step 5：C 确定性 → 汇总三个主评分Skill的信源 + 完整性校验

从产业链瓶颈、景气度、PE估值三个Skill的执行结果中提取信源等级（风险扫描器为风险调整项，不参与C乘数计算，其结论在风险聚合中引用）：

| 信源组合 | C1 |
|:---|:---:|
| 产业链瓶颈skill T1 + PE skill [T1]双验证 | 5 |
| 产业链瓶颈skill T1 + PE skill [T2] | 4 |
| 单一T1来源 | 3 |
| T2/T3混合 | 2 |
| 单一WebSearch | 1 |

C2关键假设验证率 = 产业链瓶颈skill中量化缺口验证状态 + PE skill中增速持续性确认状态，取平均。

C = (C1+C2)/2 → 映射乘数：≥8→×1.0 / 6-7→×0.9 / 4-5→×0.75 / 2-3→×0.50 / ≤1→×0.30

### Step 5.5：🆕 子Skill覆盖完整性检查（v4.9）

**核心逻辑**：窄门筛股作为首席裁决官，必须确认所有子Skill都已被调用并返回有效结果。缺失任何子Skill的评估，综合分的可靠性都会下降。

**检查清单**：

## 核心原则

| # | 原则 |
|:---|:---|
| 1 | **只找买点，不管持仓** |
| 2 | **不重复造轮子** — B找产业链skill，阶段找景气度skill，G+PE+财务健康找PE skill，风险画像找个股风险扫描器 |
| 3 | **委派但不照搬** — 每个Skill给出的评分映射到窄门筛股的0-10统一标尺 |
| 4 | **G权重最高(40%)** — 预期差=赚钱空间 |
| 5 | **不禁筛不替老大做主** — B≤2或催化剂=0标注风险但保留排序 |
| 6 | **C是乘数不是维度** |
| 7 | **黄金赔率自动加分** |
| 8 | **实时行情必查** — 没有实时股价的PE评估是算命 |
| 9 | **大事件必做映射** — S级信号必须推导受益标的，不让老大利好来了不知道该买谁 |
| 10 | **涨幅风险必标** — 追高不追高让老大自己看，但风险必须明确标注 |
| 11 | **前5名强制调用个股风险扫描器** — 必须调用@skill:"个股风险扫描器"，禁止窄门筛股自己写风险扫描逻辑 |
| 12 | **不做全市场科创拥挤度** — 该指标不纳入v4.3；情绪通过综合分+风险标注体现 |
| 13 | **综合审判八维化** — v4.9裁决段必须综合宏观/产业/B/G/C/PE/财务健康/涨幅/风险聚合/催化剂日历 |

| 15 | **事件映射必须反向验证** — 事件映射追加的标的必须至少1个T1/T2来源验证，防止变成题材联想 |

| 17 | **发布前必自检** — MD写入后自动运行完整性检查，章节/占位符/表格/两融/数据源/风险聚合全通过才交 |
| 18 | **自检通过后自动生成HTML** — 自检全绿后自动基于MD生成深色主题HTML网页，双文件一起交付 |
| 19 | **🆕 子Skill覆盖必检查** — 生成报告前强制确认4个子Skill全部返回，关键字段缺失标红 |
| 20 | **🆕 数据一致性必校验** — 跨Skill比对同一标的关键数据，差异>10%标红 |
| 21 | **🆕 风险聚合必输出** — 必须汇总所有子Skill风险标签，输出风险聚合矩阵 |
| 22 | **🆕 敏感度分析必做** — 对Top5做B/G/C权重±5%敏感性分析，排名不稳定者标注 |
| 23 | **🆕 风险调整评级必给** — 综合分相同但风险多者降级，风险直接参与排序决策 |
