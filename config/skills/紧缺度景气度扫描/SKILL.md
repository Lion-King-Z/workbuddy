## 扫描流程

### Step 1: 搜索当日内容
```bash
cd ~/.workbuddy/skills/ima-skill
node ima_api.cjs "openapi/wiki/v1/search_knowledge" '{"query":"<MMDD>","knowledge_base_id":"Y11UpEfiPNaF5OkYKkUIfkFFx2RvQwmwMANh9Ic3wV0=","cursor":""}'
```

### Step 2: 筛选紧缺相关文档
从搜索结果标题中筛选含以下关键词的DOCX文档：
- 紧缺、稀缺、涨价、供需、供给、供应、TC、库存见底、产能、景气度、资源品、材料、存储、DRAM、MLCC、CCL、PCB、铜箔、电子布、金属、稀土、钨、钾肥、六氟化钨、油运、铜矿、铝、设备、光纤、预制棒

### Step 3: 批量读取DOCX原文
用 `mcp__ima-mcp__fetch_media_content` 批量读取筛选出的DOCX文件内容。

### Step 4: 提取紧缺信号并分级

| 级别 | 标准 |
|------|------|
| **S级** | 供给刚性（全球仅1-2家供应商）、价格暴涨>30%、TC极端负值、交期>3个月 |
| **A级** | 紧缺持续且边际强化、涨价函/涨价预期明确、卖方密集推荐 |
| **B级** | 结构性紧缺、补涨逻辑、政策驱动供给收缩 |
| **C级** | 底部关注、成本支撑区、边际改善信号 |

...

### 🟡 B级：结构性紧缺/涨价预期
...

### Step 6: 渲染视觉看板 (可选)

用 `show_widget` 将Step 5的内容渲染为HTML视觉看板，复现「供应紧缺信号全景」UI风格：

**UI规范：**
- 顶部：🔥 红色方形图标 + 「供应紧缺信号全景」标题 + 信源标签
- 分级筛选标签：S/A/B/C四色圆角标签（红/橙/蓝/绿）
- 更新时间戳：绿色小圆点+实时时间
- 每个分级区域：彩色背景标题栏（S红/A橙/B蓝/C绿）+ 对应色块字母图标 + 品种计数
- 表格列：# / 品种（+日期徽章灰色小标签） / 核心信号（新信号加「新」彩色角标） / 关键数据 / 最稀缺标的（红色加粗）
- 品种颜色：S级红色、A级橙色、B级蓝色、C级绿色
- 底部：💡 核心判断区域

**设计规范：**
- 背景：`var(--color-background-primary)`，圆角卡片 `var(--border-radius-lg)`，0.5px边框
- 禁用渐变色/阴影/毛玻璃效果
- S级红色系：`#FCEBEB` 背景 / `#E24B4A` 强调色 / `#A32D2D` 文字色
- A级橙色系：`#FAEEDA` 背景 / `#EF9F27` 强调色 / `#854F0B` 文字色
- B级蓝色系：`#E6F1FB` 背景 / `#378ADD` 强调色 / `#185FA5` 文字色
- C级绿色系：`#EAF3DE` 背景 / `#639922` 强调色 / `#3B6D11` 文字色
- 字体两档：400（正文）和500（标题/强调），无600/700
- 字号：标题15px，分级标题13px，表格12px，徽章/角标10-11px
- 禁用emoji（除🔥图标外），用CSS shapes代替
- 表格 `table-layout: fixed` 防止溢出
