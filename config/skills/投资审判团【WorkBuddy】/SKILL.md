# 投资审判团【WorkBuddy】— 三分析师+风险扫描器深度研报生成器 v2.0

> **版本说明**：v2.0 在 v1.3 基础上做三项核心升级：
> 1. **新增第四个底层 Skill 调用**：`个股风险扫描器 v2.0`——从3分析师扩展为"3分析师+1风险扫描器"四维体系，14维风险画像直接嵌入研报
> 2. **引用最新底层 Skill 版本**：产业链瓶颈投研 v4.1（反向验证量化）/ 景气度 v1.4 / pe-valuation-analyst v2.5（含财务健康评估）/ 个股风险扫描器 v2.0
> 3. **信源质量汇总**：调用信源检查 v3.1 自动化触发接口，输出末尾必含信源质量汇总表
>
> 原有编排层+裁决层+HTML 输出层架构不变。首席裁决官只做四件事：调用四大底层 Skill → 四维交叉验证/冲突仲裁 → 风险聚合与仓位推导 → 输出最终评级/目标价/仓位/完整 HTML 研报。

## 一、核心承诺

给定一个标的（含代码），本 Skill 全自动完成：

`调用景气度skill → 调用产业链瓶颈投研skill → 调用pe-valuation-analyst → 调用个股风险扫描器 → 四维交叉验证 → 首席裁决 → 生成自包含 HTML 深度研报`

最终产物：
- 对话内 Markdown 摘要
- 本地 HTML 深度研报：`deliverables/investment-tribunal/<股票代码小写>-深度研报-<YYYY-MM-DD>.html`
- 自动同步 IMA 聚宝盆

## 二、本 Skill 的边界（铁律）

| 本 Skill 做什么 | 本 Skill 不做什么 |
|:---|:---|
| 调用四个底层 Skill 完成专业分析 | ❌ 不自己写景气度/瓶颈/PE/风险 的子报告 |
| 把四份子报告组装成一份 HTML 深度研报 | ❌ 不替代底层 Skill 的计算逻辑 |
| 做四维交叉验证、冲突仲裁、风险聚合、最终裁决 | ❌ 不凭空生成数据、不捏造信源 |
| 统一输出模板、字段、CSS 样式 | ❌ 不改变底层 Skill 的专业方法论 |
| 调用信源检查 v3.1 输出信源质量汇总 | ❌ 不自行判定信源等级 |

**首席裁决官 = 报告主编 + 终审法官**，不是分析师。

## 四、审判工作流 — 只做编排与裁决

### Round 1：并行调用四大底层 Skill（必须真实执行）

把用户给出的标的（代码+名称）**同时**传给四个底层 Skill，要求它们按各自输出模板返回完整章节内容。

**调用参数规范**：

```
输入：
- 标的代码：[如 688661]
- 标的名称：[如 和林微纳]
- 报告日期：[YYYY-MM-DD]
- 信源体系：T0-T4（强制）
- 输出格式：Markdown 章节，严格按各自模板字段

输出要求：
- 必须包含所有固定字段表格
- 每个定量数据必须标注 T0-T4 信源等级 + 来源 + 日期
- 必须给出明确方向：看多/看空/观望
```

三位分析师独立工作，互不通气，各自调用自己的 Skill。风险扫描官同时执行14维风险扫描。

🔴 **CHECKPOINT — Round 1 出口**：
- 收到 `景气度skill` 完整输出
- 收到 `产业链瓶颈投研skill` 完整输出（含第六章反向验证量化评分）
- 收到 `pe-valuation-analyst` 完整输出（含 Step 4.5 财务健康评估）
- 收到 `个股风险扫描器` 完整输出（含14维风险画像 + 综合风险评级 + 风险扣分）
- 四份子报告都带 T0-T4 信源标注
- **缺任意一份，不得进入 Round 2**

### Round 2：交叉质证（基于四份子报告，首席裁决官组织）

首席裁决官不创造新分析，只基于四份子报告互相质证。必须逐条输出表格：

| 挑战方 | 被挑战方 | 挑战要点 | 被挑战方回应（引用原文） | 是否导致评分调整 |
|:---:|:---:|:---|:---|:---:|
| 瓶颈 | 景气度 | 如果瓶颈被绕过/替代技术突破，订单能落地吗？ | ... | 是/否 |
| 瓶颈 | PE | 如果PE已透支瓶颈红利，瓶颈成立也消化不了估值 | ... | 是/否 |
| 景气度 | 瓶颈 | 如果产业阶段倒退，瓶颈再硬也没用 | ... | 是/否 |
| 景气度 | PE | 如果EPS下修，PE消化路径还成立吗？ | ... | 是/否 |
| PE | 瓶颈 | 认知偏差说未定价，但PEG>2.0已price in | ... | 是/否 |
| PE | 景气度 | 产业阶段定性判定是否与催化兑现进度一致？ | ... | 是/否 |
| 🆕风险扫描官 | PE | 财务健康评估发现的偿债/现金流风险，PE消化路径是否已反映？ | ... | 是/否 |
| 🆕风险扫描官 | 瓶颈 | 质押/商誉/解禁等风险是否会影响瓶颈标的的持仓安全？ | ... | 是/否 |
| 🆕风险扫描官 | 景气度 | 客户集中度/供应商集中度风险是否会影响产业阶段判断？ | ... | 是/否 |

### Round 3：四维交叉验证（首席裁决官汇总）

首席裁决官从四份子报告中提取关键结论，填入四维交叉验证记录表格：

| 维度 | 结论 | 方向 | 权重 |
|:---|:---|:---:|:---:|
| 景气度（陆远舟） | 引用景气度skill结论 | 📈/📉/— | 35% |
| 瓶颈（猎手） | 引用瓶颈skill结论（含反向验证修正后B分） | 📈/📉/— | 35% |
| 估值（PE分析师） | 引用PE skill结论（含财务健康标签） | 📈/📉/— | 20% |
| 🆕风险（风控官） | 引用风险扫描器综合风险评级 | 📈/📉/— | 10% |

#### 矩阵落格规则

根据景气度/瓶颈二维 + 估值高低，落入 9 格矩阵：

| 景气+瓶颈 | 估值低 | 估值中 | 估值高 |
|:---|:---|:---|:---|
| **双看多** | 强共振 · 买入 | 中共振 · 持有 | 弱共振 · 减仓 |
| **一多一空** | 观察 · 逢低买 | 观望 | 卖出 |
| **双看空** | 等待 · 不抄底 | 卖出 | 强烈卖出 |

**估值高低判定**（直接引用 pe-valuation-analyst 的安全边际评估）：
- 估值低：当前价 ≤ 基准估值 × 1.2
- 估值中：基准估值 × 1.2 < 当前价 ≤ 基准估值 × 3.0
- 估值高：当前价 > 基准估值 × 3.0

### Round 4：最终裁决（首席裁决官）

#### Step 1：一票否决检查

| 否决条件 | 含义 | 触发后处理 |
|:---|:---|:---|
| 景气度=L0/L6 | 产业阶段极早期或衰退期 | 直接 D 级 |
| 瓶颈 < 2.5 | 无供给壁垒 | 直接 D 级 |
| PE 溢价 > 500% 且无短期催化 | 严重透支 | 直接 D 级 |

### 5.3 强制规则

- T0 与 T1-1/T1-2/T1-4/T1-5 为**结论级信源**，可直接用于投资结论。
- T1-3（快讯）为**数据级信源**，重要数据需 T1-2/T2 交叉验证。
- T2 为**数据级信源**，可直接用于行情、财务、估值数据。
- T3 为**参考级信源**，需与 T0/T1/T2 交叉验证后使用。
- T4 为**线索级信源**，**禁止写入结论**，必须反向验证升级为 T0-T2 后方可使用。
- 所有关键判断必须 `✅双重验证`（≥2个独立信源，且至少一个 T0/T1/T2）。

### 5.4 交叉验证规则

| 结论类型 | 最低信源要求 |
|:---|:---|
| 直接写入结论 | ≥1个 T0/T1-1/T1-2；或 ≥2个独立 T1-3/T2/T3 |
| 紧缺/涨价/订单判断 | ≥1个 T1 + ≥1个 T2（独立来源） |
| 定量数据引用 | 至少 T2 及以上，且数据在有效期内 |
| 估值参数（PE/PS/增速） | ≥2个独立 T2/T3，或 1个 T2 + 1个 T3 |

**独立性判定**：同一媒体的多个报道只算 1 个独立信源；同一事件的公司公告 + 财联社报道 = 2 个独立来源。

### 5.5 时效性衰减规则

| 数据类型 | 有效期 | 过期后处理 |
|:---|:---|:---|
| 年报/半年报 | 6 个月 | 仍为 T0，但需标注报告期并检查更新 |
| 季报/公告 | 3 个月 | 仍为 T0，但需检查后续更新 |
| 产能/价格/交期数据 | 1 个月 | 降级一档，需重新确认 |
| 快讯/电报 | 1 周 | 过期需重新确认 |
| 券商研报一致预期 | 3 个月 | 过期需检查预期修正 |
| 行业数据（IDC/Yole 等） | 6 个月 | 过期需标注报告期 |
