# 每日投研推送

从多源信息流中提取10-20条真正值得注意的信号。是汇总，不是汇总。每条必须回答：知道了这个，判断会变吗？

## 采集优先级（v3.0 — T0-T4 子分级 + 聚宝盆去重 + WebSearch分级搜索）

```
Step 0 — P0.5 聚宝盆预查询
  ├─ 搜昨天电话会 + 上次推送的输出标题
  ├─ 提取已覆盖信号清单 → 去重
  └─ 目的：知道"什么已经分析过"，只推增量

Step 1 — P0 浑水调研
  └─ 限定增量标题信号，不读全文，只抓新方向

Step 2 — P1 WebSearch（分级搜索 ⚠️ 强制，按 T0-T4 信源等级）
  ├─ 第一级 allowed_domains=["cls.cn","stcn.com","cs.com.cn","eastmoney.com","cninfo.com.cn","sse.com.cn","szse.cn","jiweinet.com"]
  │   └─ T0(巨潮/交易所) / T1-2(财联社深度/证券时报/中证报/上证报) / T1-4(集微网) / T2-1(东方财富) — 可直接引用
  ├─ 第二级 不加domain，搜财联社电报/华尔街见闻/券商研报/第三方研究（T1-3/T3-2）
  └─ 第三级 兜底（T4-3 线索）：百家号/搜狐/网易 → ⛔ 仅作线索，不可直接入结论。必须回溯至≥T2 方可引用

Step 3 — P2 36氪/华尔街见闻
  └─ 外资行纪要，与浑水源对照

Step 4 — P3 雪球/股吧
  └─ 情绪拐点（仅线索）
```

每条信号标注状态：
- 🆕 新信号（聚宝盆未覆盖）
- 📌 昨日追踪（存量信号的状态更新）
- 🔗 电话会已覆盖（引用昨日分析，不重述逻辑）

### Token节约机制
- 聚宝盆查询=1-2次API（搜标题不搜正文）→ 确认已覆盖→跳过重采
- 电话会已深挖的方向→推送只提一句"昨日电话会已分析，今日XX"，不重述行业背景
- 紧缺度日报已标注的信号→推送不再独立采集WebSearch验证

### ⛔ IMA get_media_info 配额保护（2026-07-01 对齐信源 v3.0）
- 硬上限：单次任务最多10次 get_media_info
- title+highlight_content 预过滤后只对 Top 5-8条调 get_media_info
- 220021→立即停止，日报标注"IMA配额已满"，余下转 WebSearch 分级搜索（按 T0-T4 定级）
- IMA 原文可查看【浑水调研】内容视为 T1-1，配额耗尽时不降级，仅标注"待 IMA 原文验证"

## 依赖声明
本 Skill 依赖以下外部能力，缺失任一项将降级执行：

| 依赖 | 用途 | 不可用时 |
|------|------|---------|
| WebSearch | Step 1 多维度搜索采集 | 无法执行核心任务，报告网络异常 |
| 文件系统写入 | Step 5a 本地沉淀 | 跳过本地，继续IMA同步 |
| ima-skill (knowledge-base) | Step 5b IMA同步 | 跳过IMA，不阻塞本地沉淀 |

## 执行步骤

### Step 1: 平行搜索
用 WebSearch 同时采集各层级信息源。搜索关键词按当日日期替换 M月D日。

**⚠️ 分级执行**：每个维度先用 `allowed_domains=["cls.cn","stcn.com","cs.com.cn","eastmoney.com","cninfo.com.cn"]` 搜第一级（T0/T1），结果 < 3条时放宽到第二级（T2/T3 不加 domain），仍不足再兜底（T4 仅标记不引用）。

| 维度 | 搜索关键词 | 最低信源等级 |
|------|-----------|:---:|
| 宏观/政策 | "央行 利率 美联储 政策 M月D日" | T1-2 |
| AI/科技 | "半导体 AI 芯片 算力 M月D日 最新" | T1-2 |
| 资金/龙虎榜 | "龙虎榜 资金流向 北向 M月D日" | T2-1 |
| 全球/地缘 | "原油 黄金 地缘 M月D日" | T1-2 |
| 外资行电话会 | "JPM 高盛 摩根 大摩 Citi 电话会 纪要 专家会 M月D日" | T3-3 |
| 产业/供需 | "TrendForce 产能 供需 涨价 2026年7月" | T3-1 |
| 兜底宽搜 | 同上关键词但**不加 domain**（仅标记，不直接引用） | T4-3 |

### Step 2: 过滤噪音
排除：
- 纯涨跌描述
- 分析师预测/目标价
- "有望""或将""值得关注"类模糊表述
- 同一消息重复转发
- 股评/荐股/广告
- 36氪/华尔街见闻的非纪要类文章（创业报道、行业分析、政策解读等——仅保留"电话会纪要""专家会解读"类内容）

### Step 3: 优先提取（信号）

| 信号类型 | 说明 |
|----------|------|
| 供需突变 | 产能停产、发货延迟、订单暴增/骤降 |
| 价格异动 | 原材料/产品跳涨/跳跌 >5% |
| 政策超预期 | 不在市场预期内的新政策/新限制 |
| 机构行为异常 | 社保/外资/知名游资反常大买/大卖 |
| 情绪拐点 | 东方财富/百度股吧讨论热度突然翻倍或归零 |
| 产业数据交叉验证 | 不同来源对同一品种供需判断出现分歧或一致性共振 |
| 公告异动 | 重组/减持/解禁/业绩暴增 |
| 外资行信号 | 外资行电话会中海外产业链供需变化、产能调整、定价权转移 |

> 凡引用浑水调研原文（P0）的信号，标注「原文可查看[浑水调研]」。

### Step 4: 输出格式

格式：
```
# 每日信息推送 — YYYY-MM-DD（星期X）

> 概述段落：用200-300字概括今日/当周末市场核心事件、指数/盘面、重要板块、关键数据，让阅读者一眼抓住主线。

## 🔴 S级

### S1 · 标题

**核心逻辑：** 发生了什么+关键数字/幅度+直接因果（约150-200字）。

**信号意义：** 这件事改变了什么？对持仓/市场/产业链的传导（3-4点）。

**核心原因：** 为什么发生？（可选，S级关键信号必须分析深层原因）。

**催化剂链：** ▸ 近期（1-2周） ▸ 中期（1-3月） ▸ 远期（3-6月）（可选）。

**📌 昨日追踪：** 如果是对昨日信号的更新，标注此段（可选）。

**来源：** ...（信源等级：T0/T1-1/T1-2/T1-3/T1-4/T2-1/T3-3/T4-3） | 关联：XXX
```

A级、B级同上，编号为A1/A2...、B1/B2...。B级可精简字段，保留核心逻辑+信号意义/来源即可。

信源等级标注规则（⛔ 强制，T0-T4 子分级体系，与信源检查 v3.0 统一）：
- T0：巨潮/交易所公告原文 / 公司年报/一季报/问询回复
- T1-1：IMA 原文可查看【浑水调研】（43K+ 条可溯源原文）
- T1-2：财联社深度/证券时报/上海证券报/中国证券报/公司官方 IR
- T1-3：财联社电报/快讯（数据级，重要数据需 T1-2/T2 交叉验证）
- T1-4：集微网等行业垂直媒体深度
- T2-1：东方财富/同花顺/Wind/通达信（数据级，行情/财务数据可直接引用）
- T3-1：第三方产业研究（IDC/Yole/TrendForce/BloombergNEF）
- T3-2：券商研报
- T3-3：机构电话会纪要/专家访谈
- T4-1：雪球/股吧（仅情绪线索）
- T4-3：百家号/搜狐/网易 AI 聚合（⛔ 不可直接入结论，必须已交叉验证回溯至≥T2）

按信号强度排序（S→A→B），总计10-20条。

### Step 5: 沉淀
推送完成后，将完整信号内容保存到本地并同步IMA知识库。两步独立，任一步失败不阻塞另一步。

**5c. ⛔ 双端预检（5a+5b 之前强制执行，先算对再动笔）**

**IMA API 无法删除文件。上传到错误文件夹后不可逆。必须先锁定正确目标路径，再写文件和上传。**

```
🔍 双端预检 — YYYY-MM-DD_每日信息推送.md
1. 从"YYYY-MM-DD"计算正确周文件夹 MMDD-MMDD
2. 确认本地目标路径 = 深度研究/<MMDD-MMDD>/
3. 确认 IMA 目标 folder_id = 深度研究/<MMDD-MMDD> 的 folder_id
4. 预检通过 → 允许执行 5a 和 5b
```
不一致 → 🛑 阻断，修正目标变量，绝不将错就错。

**5a. 本地沉淀**
写入 `深度研究/MMDD-MMDD/YYYY-MM-DD_每日信息推送.md`，采用上述带标题、日期、概述、S1/A1/B1编号、字段结构化格式。
✅ 写入后验证：确认文件存在且大小 > 0。

**5b. IMA同步**
用 ima-skill knowledge-base 模块上传md文件至聚宝盆 → 深度研究/对应周文件夹。
流程：`preflight-check → check_repeated_names → create_media → COS upload → add_knowledge(folder_id=周文件夹ID)`。
✅ 同步后验证：每步 API 返回 code=0，最终 add_knowledge 返回 media_id 非空。
▶ 若文件夹不存在，先逐级定位/创建：聚宝盆 → 深度研究 → 对应周文件夹。

## 约束
- 必须写总标题、日期、概述（方便后续快速回溯和IMA检索）
- S/A级信号用S1/S2、A1/A2编号，B级可省略编号但保留结构化字段
- S级说清因果，B级扼要，信息密度拉满
- 没有足够信号时宁可少于10条，不凑数
- 触发词命中立刻跑，不确认不询问
- ⛔ 每条信号必须标注信源等级（T0-T4 子分级）
- ⛔ T4-3（百家号/搜狐/网易 AI 生成）入结论前必须交叉验证替换为 ≥T2 信源

## 失败模式

| 场景 | 处理 |
|------|------|
| WebSearch 返回结果不足 | 放宽关键词范围，扩大搜索区间 |
| 某信息源无法访问 | 用同层级其他源替代，不中断整体采集 |
| 所有信息源同时无法访问 | 向用户报告网络异常，不推送空内容 |
| 当日信号不足10条 | 降级到B级填充，不可臆造信号 |
| 多源信息互相矛盾 | 标注分歧，指出主流判断方向 |
| S级信号超过5条 | 收紧S级标准，只留真正影响持仓的 |
| 沉淀-本地写文件失败 | 磁盘满/权限不足/路径不存在 | 向用户报告失败环节，跳过本地沉淀，继续IMA同步 |
| 沉淀-IMA同步失败 | 网络异常/API错误/认证失效 | 向用户报告失败+错误码，不阻塞本地沉淀 |

## 反例黑名单（不要做）

| # | 反模式 | 后果 | 正确做法 |
|---|--------|------|---------|
| 1 | 把新闻标题当信号 | 信息密度低，无判断 | 必须提炼"这件事改变了什么" |
| 2 | 分析师预测/目标价进推送 | 卖方噪音，误导决策 | 只采原始数据/政策/公告 |
| 3 | S级信号过多（>5条） | 评级失效，重点=没重点 | 严格控制，仅直接影响持仓的才给 |
| 4 | 凑数硬塞B级到20条 | 稀释质量 | 宁可少于10条，不凑数 |
| 5 | 漏掉负面信号只报利好 | 单向茧房 | 正面/负面/中性按实际比例呈现 |
| 6 | 只推聊天不沉淀 | 信号散落不可追溯 | 推送后必须沉淀到本地深度研究 + IMA聚宝盆 |
