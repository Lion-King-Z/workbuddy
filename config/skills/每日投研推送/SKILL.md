# 每日投研推送

从多源信息流中提取10-20条真正值得注意的信号。是汇总，不是汇总。每条必须回答：知道了这个，判断会变吗？

## 信息源覆盖

| 层级 | 来源 | 用途 |
|------|------|------|
| 国际权威 | 彭博、路透 | 宏观转向、产业链断裂信号 |
| 外资行解读 | 36氪、华尔街见闻、选股宝 | 海外产业链前瞻、全球供需信号（JPM/高盛/MS电话会） |
| 国内快讯 | 金十数据、财联社 | 实时催化、政策发布 |
| 社区情绪 | 东方财富、百度股吧 | 热度变化、叙事转折、情绪拐点 |
| 机构数据 | 券商研报、行业协会、TrendForce | 供需/产能/价格硬数据 |
| 一手深度 | 星大派特刊等内部资料 | 独家逻辑、前瞻判断 |

## 依赖声明
本 Skill 依赖以下外部能力，缺失任一项将降级执行：

| 依赖 | 用途 | 不可用时 |
|------|------|---------|
| WebSearch | Step 1 多维度搜索采集 | 无法执行核心任务，报告网络异常 |
| 文件系统写入 | Step 5a 本地沉淀 | 跳过本地，继续IMA同步 |
| ima-skill (knowledge-base) | Step 5b IMA同步 | 跳过IMA，不阻塞本地沉淀 |

## 执行步骤

### Step 1: 平行搜索
用 WebSearch 同时采集各层级信息源。搜索关键词按当日日期替换 M月D日。

| 维度 | 搜索关键词 |
|------|-----------|
| 宏观/政策 | "央行 利率 美联储 政策 M月D日" |
| AI/科技 | "半导体 AI 芯片 算力 M月D日 最新" |
| 资金/龙虎榜 | "龙虎榜 资金流向 北向 M月D日" |
| 全球/地缘 | "原油 黄金 地缘 M月D日" |
| 外资行/海外产业链 | "高盛 摩根 大摩 Citi 电话会 纪要 研报 M月D日 36氪 华尔街见闻" |
| 产业/供需 | "TrendForce 产能 供需 涨价 2026年6月" |

### Step 2: 过滤噪音
排除：
- 纯涨跌描述
- 分析师预测/目标价
- "有望""或将""值得关注"类模糊表述
- 同一消息重复转发
- 股评/荐股/广告

### Step 3: 优先提取（信号）

| 信号类型 | 说明 |
|----------|------|
| 供需突变 | 产能停产、发货延迟、订单暴增/骤降 |
| 价格异动 | 原材料/产品跳涨/跳跌 >5% |
| 政策超预期 | 不在市场预期内的新政策/新限制 |
| 机构行为异常 | 社保/外资/知名游资反常大买/大卖 |
| 情绪拐点 | 东方财富/百度股吧讨论热度突然翻倍或归零 |
| 产业数据交叉验证 | 不同来源对同一品种供需判断出现分歧或一致性共振 |
| 公告异动 | 重组/减持/解禁/业绩暴增 |
| 外资行信号 | JPM/高盛/MS电话会中海外产业链供需变化、目标价/评级重大调整 |

### Step 4: 输出格式

格式：
```
🔴S / 🟡A / 🟢B 【维度】标题
详情（S级约200字：发生了什么+关键数字/幅度+产业链传导逻辑+为什么影响持仓；A级100-150字；B级50-80字）
来源：... | 关联：XXX
```

按信号强度排序（S→A→B），总计10-20条。

### Step 5: 沉淀
推送完成后，将完整信号内容保存到本地并同步IMA知识库。两步独立，任一步失败不阻塞另一步。

**5a. 本地沉淀**
写入 `深度研究/MMDD-MMDD/YYYY-MM-DD_每日信息推送.md`，按S→A→B排序，去掉emoji，用标题式结构。
✅ 写入后验证：确认文件存在且大小 > 0。

**5b. IMA同步**
用 ima-skill knowledge-base 模块上传md文件至聚宝盆 → 深度研究/对应周文件夹。
流程：`preflight-check → check_repeated_names → create_media → COS upload → add_knowledge(folder_id=周文件夹ID)`。
✅ 同步后验证：每步 API 返回 code=0，最终 add_knowledge 返回 media_id 非空。
▶ 若文件夹不存在，先逐级定位/创建：聚宝盆 → 深度研究 → 对应周文件夹。

## 约束
- 不写总标题、不写日期、不写总结
- S级说清因果，B级扼要，信息密度拉满
- 没有足够信号时宁可少于10条，不凑数
- 触发词命中立刻跑，不确认不询问

## 失败模式

| 场景 | 处理 |
|------|------|
| WebSearch 返回结果不足 | 放宽关键词范围，扩大搜索区间 |
| 某信息源无法访问 | 用同层级其他源替代，不中断整体采集 |
| 所有信息源同时无法访问 | 向用户报告网络异常，不推送空内容 |
| 当日信号不足10条 | 降级到B级填充，不可臆造信号 |
| 多源信息互相矛盾 | 标注分歧，指出主流判断方向 |
| S级信号超过5条 | 收紧S级标准，只留真正影响持仓的 |
| 沉淀-本地写文件失败 | 磁盘满/权限不足/路径不存在 | 向用户报告失败环节，跳过本地沉淀，继续IMA同步 |
| 沉淀-IMA同步失败 | 网络异常/API错误/认证失效 | 向用户报告失败+错误码，不阻塞本地沉淀 |

## 反例黑名单（不要做）

| # | 反模式 | 后果 | 正确做法 |
|---|--------|------|---------|
| 1 | 把新闻标题当信号 | 信息密度低，无判断 | 必须提炼"这件事改变了什么" |
| 2 | 分析师预测/目标价进推送 | 卖方噪音，误导决策 | 只采原始数据/政策/公告 |
| 3 | S级信号过多（>5条） | 评级失效，重点=没重点 | 严格控制，仅直接影响持仓的才给 |
| 4 | 凑数硬塞B级到20条 | 稀释质量 | 宁可少于10条，不凑数 |
| 5 | 漏掉负面信号只报利好 | 单向茧房 | 正面/负面/中性按实际比例呈现 |
| 6 | 只推聊天不沉淀 | 信号散落不可追溯 | 推送后必须沉淀到本地深度研究 + IMA聚宝盆 |
