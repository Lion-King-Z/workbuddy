# 每日投研推送

从多源信息流中提取10-20条真正值得注意的信号。是汇总，不是汇总。每条必须回答：知道了这个，判断会变吗？

## 信息源覆盖

| 层级 | 来源 | 用途 |
|------|------|------|
| 国际权威 | 彭博、路透 | 宏观转向、产业链断裂信号 |
| 外资行解读 | 36氪、华尔街见闻、选股宝 | 海外产业链前瞻、全球供需信号（JPM/高盛/MS电话会） |
| 国内快讯 | 金十数据、财联社 | 实时催化、政策发布 |
| 社区情绪 | 东方财富、百度股吧 | 热度变化、叙事转折、情绪拐点 |
| 机构数据 | 券商研报、行业协会、TrendForce | 供需/产能/价格硬数据 |
| 一手深度 | 星大派特刊等内部资料 | 独家逻辑、前瞻判断 |

## 依赖声明
本 Skill 依赖以下外部能力，缺失任一项将降级执行：

| 依赖 | 用途 | 不可用时 |
|------|------|---------|
| WebSearch | Step 1 多维度搜索采集 | 无法执行核心任务，报告网络异常 |
| 文件系统写入 | Step 5a 本地沉淀 | 跳过本地，继续IMA同步 |
| ima-skill (knowledge-base) | Step 0 扫描+Step 5b IMA同步 | Step 0跳过（无电话会信号），Step 5b跳过本地沉淀 |
| ima-mcp (get_knowledge_list) | Step 0 扫描浑水调研新内容 | Step 0跳过，降级为纯公开信息推送 |
| 浑水电话会深度分析Skill | Step 0b 六阶段深度分析 | Step 0跳过，仅标记有新内容待手动处理 |

## 执行步骤

### Step 0: 前置扫描 — IMA浑水调研文件夹（自动化触发电话会深度分析）

**目的**：把「浑水电话会深度分析」从手动触发改为自动化触发。每日推送前先扫IMA，有新电话会就自动跑深度分析+紧缺信号全景图，结果融入当晚推送。

**执行逻辑**：

```
1. 调用 IMA get_knowledge_list(knowledge_base_id=聚宝盆, folder_id=浑水调研根folder_7471884757106786, sort_type=UPDATE_TS_DESC_SORT_TYPE, limit=20)
2. 对返回的每条记录，检查 update_ts 是否在最近3天内
3. 同时检查周子文件夹（如 0615-0621/）内是否有新上传的md文件
4. 汇总新内容清单
```

**分支判断**：

| 情况 | 动作 |
|------|------|
| 最近3天**无**新内容 | 跳过Step 0，直接进入Step 1正常推送 |
| 最近3天**有**新内容（1-3条） | 对每条新内容执行Step 0a → 0b → 0c |
| 最近3天**有**新内容（>3条） | 只取最新3条，避免单日过载；其余记入日志下次处理 |

**0a. 拉取电话会原文**

```
对每条新内容：
- 如果是 NOTE 类型 → 调用 IMA get_note_content 拉取笔记全文
- 如果是 MARKDOWN/PDF 类型 → 调用 IMA fetch_media_content 拉取文件内容
- 提取日期（从标题或文件名 YYYY-MM-DD 模式匹配）
- 提取正文文本
```

**0b. 触发浑水电话会深度分析Skill（六阶段管线）**

把拉取到的原文作为输入，调用 `浑水电话会深度分析` Skill 的完整六阶段管线：
- 阶段一：接收解析（用0a拉取的原文）
- 阶段二：瓶颈分析
- 阶段三：景气度扫描
- 阶段四：信源验证
- 阶段五：IMA沉淀（深度MD + 笔记）
- 阶段六：紧缺信号全景图生成（S/A/B分级表）

**关键约束**：
- 每条新电话会独立跑一遍完整六阶段，不合并处理
- 产出物按电话会日期归档到 `浑水调研/<周文件夹>/YYYY-MM-DD_电话会_关键词.md` + `YYYY-MM-DD_紧缺信号全景.md`
- IMA同步时严格遵守浑水Skill的落盘前预检规则（folder_id映射表）

**0c. 提取推送素材**

从阶段六产出的紧缺信号全景图中提取S/A/B级信号，作为当晚推送的**头部内容**：

```
对每条电话会的全景图：
- 提取所有S级品种 → 转为推送格式的S级信号
- 提取所有A级品种 → 转为推送格式的A级信号
- B级品种 → 不进推送，只归档（避免稀释推送质量）
- 信号来源标注：「浑水电话会 YYYY-MM-DD」
```

**Step 0 产出**：
- 0-3份完整电话会深度分析MD（存浑水调研周文件夹）
- 0-3份紧缺信号全景图（存浑水调研周文件夹）
- 推送素材池：S/A级信号列表（待Step 4融合输出）

🔴 **CHECKPOINT**：Step 0完成后，把0c提取的信号加入Step 3的信号池，再进入Step 1继续采集其他维度的公开信息。

### Step 1: 平行搜索
用 WebSearch 同时采集各层级信息源。搜索关键词按当日日期替换 M月D日。

| 维度 | 搜索关键词 |
|------|-----------|
| 宏观/政策 | "央行 利率 美联储 政策 M月D日" |
| AI/科技 | "半导体 AI 芯片 算力 M月D日 最新" |
| 资金/龙虎榜 | "龙虎榜 资金流向 北向 M月D日" |
| 全球/地缘 | "原油 黄金 地缘 M月D日" |
| 外资行电话会 | "JPM 高盛 摩根 大摩 Citi 电话会 纪要 专家会 M月D日 36氪 华尔街见闻" |
| 产业/供需 | "TrendForce 产能 供需 涨价 2026年6月" |

### Step 2: 过滤噪音
排除：
- 纯涨跌描述
- 分析师预测/目标价
- "有望""或将""值得关注"类模糊表述
- 同一消息重复转发
- 股评/荐股/广告
- 36氪/华尔街见闻的非纪要类文章（创业报道、行业分析、政策解读等——仅保留"电话会纪要""专家会解读"类内容）

### Step 3: 优先提取（信号）

| 信号类型 | 说明 |
|----------|------|
| 供需突变 | 产能停产、发货延迟、订单暴增/骤降 |
| 价格异动 | 原材料/产品跳涨/跳跌 >5% |
| 政策超预期 | 不在市场预期内的新政策/新限制 |
| 机构行为异常 | 社保/外资/知名游资反常大买/大卖 |
| 情绪拐点 | 东方财富/百度股吧讨论热度突然翻倍或归零 |
| 产业数据交叉验证 | 不同来源对同一品种供需判断出现分歧或一致性共振 |
| 公告异动 | 重组/减持/解禁/业绩暴增 |
| 外资行信号 | 外资行电话会中海外产业链供需变化、产能调整、定价权转移 |

### Step 4: 输出格式

**信号优先级排序**：浑水电话会S/A级信号（来自Step 0）> 公开采集S级 > 公开采集A级 > B级

格式：
```
🔴S / 🟡A / 🟢B 【维度】标题
详情（S级约200字：发生了什么+关键数字/幅度+产业链传导逻辑+为什么影响持仓；A级100-150字；B级50-80字）
来源：... | 关联：XXX
```

**浑水电话会信号的额外标注**：
- 来源行必须标注 `来源：浑水电话会 YYYY-MM-DD | 原文可查看【浑水调研】`
- 如果是S级紧缺信号，标题前加 `🔥` 标记，并在详情末尾追加「最稀缺标的：XXX / XXX」
- 同一晚多场电话会的信号，按电话会日期倒序排列（最新的在前）

按信号强度排序（S→A→B），总计10-20条。

### Step 5: 沉淀
推送完成后，将完整信号内容保存到本地并同步IMA知识库。两步独立，任一步失败不阻塞另一步。

**沉淀范围**：
- 每日推送主文件：`深度研究/<周文件夹>/YYYY-MM-DD_每日信息推送.md`（包含浑水电话会信号+公开采集信号）
- 浑水电话会深度分析：已在Step 0b沉淀到 `浑水调研/<周文件夹>/`，此处不重复
- 紧缺信号全景图：已在Step 0b沉淀到 `浑水调研/<周文件夹>/`，此处不重复

**5c. ⛔ 双端预检（5a+5b 之前强制执行，先算对再动笔）**

**IMA API 无法删除文件。上传到错误文件夹后不可逆。必须先锁定正确目标路径，再写文件和上传。**

```
🔍 双端预检 — YYYY-MM-DD_每日信息推送.md
1. 从"YYYY-MM-DD"计算正确周文件夹 MMDD-MMDD
2. 确认本地目标路径 = 深度研究/<MMDD-MMDD>/
3. 确认 IMA 目标 folder_id = 深度研究/<MMDD-MMDD> 的 folder_id
4. 预检通过 → 允许执行 5a 和 5b
```
不一致 → 🛑 阻断，修正目标变量，绝不将错就错。

**5a. 本地沉淀**
写入 `深度研究/MMDD-MMDD/YYYY-MM-DD_每日信息推送.md`，按S→A→B排序，去掉emoji，用标题式结构。
✅ 写入后验证：确认文件存在且大小 > 0。

**5b. IMA同步**
用 ima-skill knowledge-base 模块上传md文件至聚宝盆 → 深度研究/对应周文件夹。
流程：`preflight-check → check_repeated_names → create_media → COS upload → add_knowledge(folder_id=周文件夹ID)`。
✅ 同步后验证：每步 API 返回 code=0，最终 add_knowledge 返回 media_id 非空。
▶ 若文件夹不存在，先逐级定位/创建：聚宝盆 → 深度研究 → 对应周文件夹。

## 约束
- 不写总标题、不写日期、不写总结
- S级说清因果，B级扼要，信息密度拉满
- 没有足够信号时宁可少于10条，不凑数
- 触发词命中立刻跑，不确认不询问

## 失败模式

| 场景 | 处理 |
|------|------|
| WebSearch 返回结果不足 | 放宽关键词范围，扩大搜索区间 |
| 某信息源无法访问 | 用同层级其他源替代，不中断整体采集 |
| 所有信息源同时无法访问 | 向用户报告网络异常，不推送空内容 |
| 当日信号不足10条 | 降级到B级填充，不可臆造信号 |
| 多源信息互相矛盾 | 标注分歧，指出主流判断方向 |
| S级信号超过5条 | 收紧S级标准，只留真正影响持仓的 |
| 沉淀-本地写文件失败 | 磁盘满/权限不足/路径不存在 | 向用户报告失败环节，跳过本地沉淀，继续IMA同步 |
| 沉淀-IMA同步失败 | 网络异常/API错误/认证失效 | 向用户报告失败+错误码，不阻塞本地沉淀 |
| **Step 0 IMA扫描失败** | API错误/认证失效 | 跳过Step 0，继续Step 1正常推送，在推送末尾标注「⚠️ 今日未自动扫描浑水调研，请手动检查」 |
| **Step 0a 拉取原文失败** | 笔记/文件权限不足 | 记录失败的media_id，跳过该条，继续处理其他新内容 |
| **Step 0b 浑水深度分析异常** | 六阶段管线某阶段STOP | 该条电话会降级为B级摘要信号进推送，标注「⚠️ 深度分析未完成，仅有原文摘要」，完整分析待手动触发 |
| **Step 0 新内容>3条** | 单日过载 | 只处理最新3条，其余记入日志，次日继续处理 |

## 反例黑名单（不要做）

| # | 反模式 | 后果 | 正确做法 |
|---|--------|------|---------|
| 1 | 把新闻标题当信号 | 信息密度低，无判断 | 必须提炼"这件事改变了什么" |
| 2 | 分析师预测/目标价进推送 | 卖方噪音，误导决策 | 只采原始数据/政策/公告 |
| 3 | S级信号过多（>5条） | 评级失效，重点=没重点 | 严格控制，仅直接影响持仓的才给 |
| 4 | 凑数硬塞B级到20条 | 稀释质量 | 宁可少于10条，不凑数 |
| 5 | 漏掉负面信号只报利好 | 单向茧房 | 正面/负面/中性按实际比例呈现 |
| 6 | 只推聊天不沉淀 | 信号散落不可追溯 | 推送后必须沉淀到本地深度研究 + IMA聚宝盆 |
