# 估值是门艺术 v1.1

> **核心定位**：估值不是算 PE，而是回答“当前价格反映了什么预期”“这个预期是否合理”。
> 
> 本 Skill 是 **pe-valuation-analyst 的上层框架**：先定性定位（艺术），再调用 pe-valuation-analyst 做定量估值（科学）。

---

## 一、核心哲学

1. **PE 是结果，不是原因**。先判断公司所处阶段和定价权，再决定该给什么 PE。
2. **价格是预期的函数**。当前股价 = 市场对未来的折现。估值工作是判断这个折现是否合理。
3. **估值是概率分布，不是单点数字**。必须给出牛/基/熊三情景及概率。
4. **安全边际来自下行保护，不是低 PE**。成长股的安全边际是“最坏情况下还能值多少”。
5. **估值必须动态重估**。每季财报、每个重大订单、每次政策变化都可能导致重估。

---

## 二、工作流

```
Step 0: 赛道爆发判断 → 调用 行业爆发判断法
        → 输出：行业分级、边际方向、六维打分、分级变化信号
        → 作用：决定后续估值的基调、PE区间系数与仓位上限

Step 1: 产业阶段定位 → 调用 景气度skill
Step 2: 产业链瓶颈定位 → 调用 产业链瓶颈投研skill
Step 3: 四维度验证漏斗 → 筛选标的质量
Step 4: 三梯队分类 → 按时间/验证度分层
Step 5: 二维估值定位矩阵 → 确定估值象限与基础 PE 区间
Step 6: 调用 pe-valuation-analyst → 定量估值（T1/T2/T3、PEG、PE陷阱）
Step 7: 情景分析 → 牛/基/熊概率加权
Step 8: 风险归因 → 什么事实会打破逻辑
Step 9: 催化剂时间轴 → 市场何时重新定价
Step 10: 安全边际与仓位建议 → 是否可买、买多少
Step 11: 退出规则 → 止盈、止损、逻辑破坏退出
```

## 三、模块 0：赛道爆发判断（Step 0）

估值必须建立在赛道景气度之上。没有高景气主线的估值，容易变成“在泥潭里找金子”。因此本 Skill 在起手阶段**显式调用行业爆发判断法**，先对标的所属赛道进行量化分级，再进入公司级估值。

### 调用方式

对每个标的，先提取其所属赛道/板块/品类，然后调用 `行业爆发判断法`。

> 注意：若用户直接问“XX 公司估值”，本 Skill 应自动识别其所属赛道并调用；若用户已提供赛道分级结论，可直接复用，但仍需在工作流中展示该结论。

### 输入参数

- 赛道/板块/品类边界（如：半导体 WF6 特气、HBM 前驱体、溅射靶材）
- 样本公司（5-10 家收入暴露度高的代表公司）
- 最新财报季、订单/排产/价格数据、政策消息

### 输出字段

| 字段 | 说明 | 对估值的影响 |
|:---|:---|:---|
| **当前分级** | 爆发级 / 观察级 / 未爆发 | 决定 PE 区间基调 |
| **边际方向** | 上修中 / 下修中 / 稳定 / 待确认 | 决定催化剂权重与安全边际要求 |
| **六维打分** | 收入/加速度/广度/订单/价格/库存 | 作为赛道质量证据 |
| **分级变化信号** | 最新正/负信号及来源 | 作为风险归因输入 |

### 赛道分级 → 估值基调映射

| 赛道分级 | 估值基调 | 基础 PE 调整 | 仓位系数 | 安全边际要求 |
|:---|:---|---:|---:|---:|
| **爆发级 + 上修中** | 高景气主线，可积极给估值 | +15%~25% | 1.3x | 可适度放宽 |
| **爆发级 + 稳定** | 高景气已确认 | +10%~15% | 1.0x | 标准 |
| **观察级 + 上修中** | 景气改善中，估值需验证 | 0~+10% | 0.7x | 标准 |
| **观察级 + 稳定** | 景气平庸 | 0% | 0.5x | 更严格 |
| **观察级 + 下修中** | 景气转弱 | -10%~-20% | 0.4x | 更严格 |
| **未爆发** | 主题或局部机会 | -20%~-30% | ≤0.2x | 极严格 |

> 注：上述调整为相对“二维估值定位矩阵”基础 PE 区间的浮动。

## 四、模块 1：四维度验证漏斗

对候选标的按以下维度打分（0-5 分）：

| 维度 | 权重 | 核心问题 |
|:---|:---:|:---|
| 验证度 | 25% | 是否有订单/涨价/量产/客户导入？ |
| 现金流 | 25% | 利润是否兑现？关注 FCF 而非会计利润。 |
| 全链掌控 | 20% | 是否覆盖资源→材料→器件→终端？ |
| 分润/抢单 | 15% | 是否从他国手中拿到实际份额？ |
| 护城河 | 15% | 客户转换成本高吗？技术/认证壁垒是什么？ |

**梯队判定**：
- 第一梯队（快速反制）：综合分 ≥ 4.0，且验证度 ≥ 4
- 第二梯队（中期验证）：综合分 2.5-3.9
- 第三梯队（长期壁垒）：综合分 < 2.5

## 四、模块 2：二维估值定位矩阵

### X 轴：时间 / 验证度

| 阶段 | 特征 | 估值特征 |
|:---|:---|:---|
| 概念期 | 无订单、无验证，仅有主题 | 期权估值，高 PE 高风险 |
| 验证期 | 送样/小批量/二供，价受压 | 高 PE，关注验证里程碑 |
| 放量期 | 已通过认证，订单快速放量 | 中高 PE，业绩锚定开始 |
| 成熟期 | 市场份额稳定，增速下降 | 稳定 PE，看分红/回购 |

### Y 轴：客户锁定 / 定价权

| 层级 | 特征 | 估值特征 |
|:---|:---|:---|
| 概念/备胎 | 无认证，客户随时可切 | 大幅折价 |
| 验证导入 | 二供/送样，价格受压 | 小溢价 |
| 认证绑定 | 长协、核心份额、客户难替换 | 显著溢价 |
| 标准共建 | 参与下一代定义，共同开发 | 最高溢价 |

### 象限 → 基础 PE 区间（参考）

|  | 概念/备胎 | 验证导入 | 认证绑定 | 标准共建 |
|---|:---:|:---:|:---:|:---:|
| **概念期** | 不可用 | 50-80x | 80-120x | 100-150x |
| **验证期** | 30-50x | 40-60x | 60-90x | 80-120x |
| **放量期** | 20-30x | 30-45x | 45-70x | 60-100x |
| **成熟期** | 15-20x | 20-30x | 30-45x | 40-60x |

> 注：具体 PE 区间需结合行业增速、景气度、可比公司调整。上述为半导体/新材料主题参考区间。

### 地缘/客户场景标签（作为风险/溢价调整因子）

原始“抗日先锋/棒子共赢/国产替代/达子新品”不是 Y 轴，而是**场景标签**，用于调整估值：

| 场景标签 | 典型影响 | 调整方向 |
|:---|:---|:---|
| 抗日先锋 | 承接日本产能缺口，政策催化强 | +10%~20% 溢价 |
| 棒子共赢 | 切入韩国核心供应链，订单可见 | +15%~25% 溢价 |
| 国产替代 | 替代率低，需求迫切，但验证长 | 视验证进度 ±10% |
| 达子新品 | 新架构，高弹性但高不确定性 | 折价/溢价两极分化 |

## 六、模块 4：情景分析

每个标的必须给出三情景及概率：

| 情景 | 概率 | 假设 | 目标价 | 期望贡献 |
|:---|:---:|:---|---:|---:|
| 牛市 | P1 | 核心催化兑现，订单/涨价超预期 | T3 | P1 × (T3 - P) |
| 基准 | P2 | 催化按预期兑现，行业正常增速 | T1/T2 | P2 × (T1/T2 - P) |
| 熊市 | P3 | 验证失败/订单流失/价格战 | 下行价值 | P3 × (下行价值 - P) |

**概率加权目标价 = P1×T3 + P2×T2 + P3×下行价值**

---

## 七、模块 5：风险归因与退出

不是机械止损 -8%，而是识别“什么事实会打破投资逻辑”：

| 风险类型 | 打破逻辑的信号 | 退出动作 |
|:---|:---|:---|
| 政策风险 | 管制放松/补贴取消 | 减仓/清仓 |
| 客户风险 | 核心客户砍单/验证失败 | 清仓 |
| 技术风险 | 替代路线胜出/认证未通过 | 减仓 |
| 竞争风险 | 价格战/新进入者放量 | 减仓 |
| 周期风险 | 下游砍单/库存堆积 | 减仓 |
| 估值风险 | 当前价透支 T3 | 止盈 |

**退出触发条件**（任一触发即启动退出流程）：
1. 投资逻辑被破坏的事实出现
2. 当前价 > T3 且无新催化剂
3. 一致预期 EPS 连续 2 月下修 > 10%
4. 季报营收增速低于预期 > 20%
5. 股价跌破关键支撑位（如 T1 或 60 日线）

## 八、模块 6：催化剂时间轴

列出未来 3-6 个月的关键事件：

```
YYYY.MM │ 📌 事件1：大客户招标结果 → 若中标：加仓；若落选：减仓
YYYY.MM │ 📌 事件2：中报业绩发布 → 验证放量逻辑
YYYY.MM │ 🔑 关键窗口：产品认证通过 → 可持续性确认
```

**催化剂新鲜度衰减**：
- 首次出现：×1.0
- 次日：×0.85
- 第 3 天：×0.70
- 5 天+：×0.50

## 九、模块 7：安全边际与仓位建议

### 安全边际评估

- 当前价 vs 基准估值（T1）：溢价率
- 当前价 vs 催化估值（T3）：溢价率
- 下行价值（熊市情景）：若逻辑失败，值多少钱
- **安全边际 = 当前价到下行价值的距离 / 当前价**

### 仓位建议（风险调整后）

```
基础仓位 = 三梯队权重
  第一梯队：30-50%
  第二梯队：10-20%
  第三梯队：0-5%

↓ 调整 by 验证度（越高越重）
↓ 调整 by 流动性（日均成交额/流通市值，越小越轻）
↓ 调整 by 组合相关性（与已有持仓重合度越高越轻）
↓ 调整 by 安全边际（越大越重）

最终仓位 = 基础仓位 × 流动性系数 × 相关性系数 × 安全边际系数
```

> 80/15/5 的固定比例过于机械，建议用上述风险调整法替代。

## 十、输出格式

必须输出以下板块：

1. **估值定位摘要**：一句话结论（高估/合理/低估/严重低估）
2. **赛道景气度输入卡**：行业分级、边际方向、六维打分、分级变化信号
3. **二维定位矩阵**：X 轴、Y 轴、象限、基础 PE 区间（已叠加景气度系数）
4. **四维度验证打分表**：5 个维度分数 + 梯队判定
5. **pe-valuation-analyst 定量估值结果**：引用其估值摘要表
6. **情景分析**：牛/基/熊三情景 + 概率加权目标价
7. **风险归因**：核心风险点 + 退出触发条件
8. **催化剂时间轴**：未来 3-6 个月关键事件
9. **安全边际与仓位建议**：当前价 vs 各目标价 + 建议仓位
10. **动态重估提醒**：下次重估时间与触发条件

## 十一、禁止行为

| # | 禁止项 | 原因 |
|---:|---|---|
| 1 | 跳过赛道爆发判断直接定位估值 | 无景气度前提的估值是空中楼阁 |
| 2 | 跳过二维定位直接算 PE | 缺乏定性判断的 PE 是数字游戏 |
| 3 | 把地缘标签当作 Y 轴 | 地缘是场景，不是估值基础 |
| 4 | 只给一个目标价，无情景分析 | 估值是概率分布 |
| 5 | 只有止损，没有逻辑破坏退出 | 机械止损不等于风险管理 |
| 6 | 忽略催化剂时间轴 | 不知道何时重新定价 |
| 7 | 忽略流动性约束 | 小盘股无法重仓 |
| 8 | 使用 T4 信源做估值判断 | 可信度不足 |
| 9 | 不调用 pe-valuation-analyst 做定量校验 | 定性结论需定量验证 |
