# 项目结构规范

锅师投研项目的文件组织规则。

## 根目录结构

```
F:\workbuddy\锅师\        ← 公司电脑路径
G:\锅师\                   ← 家用电脑路径

├── 深度研究/               ← 产业链深度研报专用
│   ├── MMDD-MMDD/         ← 周文件夹（按产出日期归入）
│   │   └── YYYY-MM-DD_主题深研.md
│   └── YYYY-MM_月报.md
├── 浑水调研/               ← 浑水生态分析专用
│   ├── MMDD-MMDD/         ← 周文件夹（同上）
│   │   ├── 电话会_*.md
│   │   ├── 浑水研报紧缺总结_*.md
│   │   └── 供应紧缺度分析_*.md
│   ├── 浑水跟踪_YYYY-MM.md
│   ├── 紧缺度追踪总表_YYYYMMDD.md
│   └── 紧缺信号时间线_*.html
├── 信息源清单.md
├── 年报_YYYY.md
├── 聚宝盆索引.md
└── .workbuddy/             ← WorkBuddy 系统文件

F:\个人成长\                 ← 独立文件夹（在项目外）
├── YYYYWww_个人成长.md
└── YYYY-MM_个人成长.md
```

## 核心原则

1. **2目录+N文件**：深度研究/ + 浑水调研/ + 根目录独立文件
2. **深度研究 = 产业链**：仅放产业链深度研报，不放电话会、日报等
3. **浑水调研 = 浑水生态**：电话会、景气度扫描、紧缺度分析、月度跟踪——凡浑水相关全进此处
4. **时间容器 > 类型容器**：按产出日期归入周文件夹（MMDD-MMDD/），不是按文件类型分文件夹
5. **跨周期追踪表放根目录**：紧缺度追踪总表、时间线等跨周期文件放根目录，不拆入周文件夹
6. **个人成长在项目外**：`F:\个人成长\` 独立管理，不在锅师项目内
7. **无形式化产出**：不生成日报/简报/研报，只产出电话会深度分析

## 工作流

当用户问「文件放哪里」「这个文件存哪里」「创建新文件」时，按以下流程：

```
1. 判断文件类型 → 产业链研报 / 浑水生态 / 跨周期追踪 / 根目录独立 / 个人成长
2. 判断产出日期 → 计算所属周文件夹（MMDD-MMDD）
3. 确认当前机器 → F盘（公司机）或 G盘（家用机）
4. 输出完整路径 → 创建目录（若不存在）+ 写入文件
```

🔴 **CHECKPOINT**：输出路径前必须确认：
- [ ] 文件类型已明确（5类之一）
- [ ] 周文件夹计算正确（周一~周日为一周）
- [ ] 当前机器盘符已识别（F/G）

🛑 **STOP**：若文件类型无法明确判断，停止输出路径，先向用户确认文件内容性质。

## 异常与边界条件

| 触发条件 | 一线修复 | 仍失败兜底 |
|----------|----------|-----------|
| 周文件夹不存在 | `mkdir -p` 创建完整路径 | 标注「目录创建失败，请手动创建」 |
| 跨周期文件被误放入周文件夹 | 移动到根目录 + 提醒用户 | 保留原位置 + 标注「应属根目录」 |
| 双机路径不一致（F盘 vs G盘） | 询问当前机器 → 替换盘符 | 默认F盘 + 标注「G盘需手动同步」 |
| 产出日期跨周（如周日产出归本周还是下周） | 按产出当日归入当周（周日归当周） | — |
| 文件类型边界模糊（如浑水产业链深度研报） | 浑水相关优先归「浑水调研/」 | 询问用户偏好 |
| 个人成长文件被误放入锅师项目 | 移动到 `F:\个人成长\` 或 `G:\个人成长\` | 标注「应属项目外」 |
| 月报/年报类型 | 月报放「深度研究/」根目录，年报放项目根目录 | — |

## 禁止行为（反例黑名单）

| # | 禁止项 | 原因 | 正确做法 |
|---|--------|------|----------|
| 1 | 跨周期文件（紧缺度追踪总表/时间线）放入周文件夹 | 破坏跨周期追踪完整性 | 放入「浑水调研/」根目录 |
| 2 | 个人成长文件放入锅师项目 | 混淆项目边界 | 放入 `F:\个人成长\` 或 `G:\个人成长\` |
| 3 | 日报/简报/研报等形式化产出存入项目 | 违反「无形式化产出」原则 | 只产出电话会深度分析 |
| 4 | 浑水相关文件放入「深度研究/」 | 违反「2目录分工」原则 | 浑水生态全入「浑水调研/」 |
| 5 | 产业链研报放入「浑水调研/」 | 违反目录分工 | 产业链研报入「深度研究/」 |
| 6 | 周文件夹命名用「YYYY-MM-DD」格式 | 违反命名规范 | 用「MMDD-MMDD」起止日期 |
| 7 | 跳过盘符确认直接输出路径 | 双机不一致会写错位置 | 先确认F/G盘再输出 |

## 全局检查点规则

工作流四步必须按顺序执行，不可跳过。🛑 STOP 触发时必须向用户确认文件类型后再继续。

## 文件命名规则

| 类型 | 命名格式 |
|------|---------|
| 产业链深度研报 | `YYYY-MM-DD_主题深研.md` |
| 电话会分析 | `YYYY-MM-DD_电话会_关键词.md` |
| 紧缺度分析 | `YYYY-MM-DD_供应紧缺度分析.md` 或 `紧缺度追踪总表_YYYYMMDD.md` |
| 月度跟踪 | `浑水跟踪_YYYY-MM.md` |
| 月报 | `YYYY-MM_月报.md` |
| 年报 | `年报_YYYY.md` |
| 个人成长 | `YYYYWww_个人成长.md` 或 `YYYY-MM_个人成长.md` |

## 周文件夹规则

- 按产出日期归入对应周
- 周文件夹命名：`MMDD-MMDD`（起止日期）
- 例如 6月15日的产出归入 `0615-0621/`
- 周一~周日为一周

## 路径切换

在G盘（家用机）和F盘（公司机）之间切换时，直接替换盘符和前缀：
- 家用机：`G:\锅师\`
- 公司机：`F:\workbuddy\锅师\`
- 个人成长：家用机 `G:\个人成长\`，公司机 `F:\个人成长\`

### 切换命令示例

**公司机创建周文件夹**：
```bash
mkdir -p "F:/workbuddy/锅师/深度研究/0615-0621"
mkdir -p "F:/workbuddy/锅师/浑水调研/0615-0621"
```

**家用机创建周文件夹**：
```bash
mkdir -p "G:/锅师/深度研究/0615-0621"
mkdir -p "G:/锅师/浑水调研/0615-0621"
```

**判断当前机器**：
```bash
# 检查F盘是否存在（公司机）
test -d "F:/workbuddy/锅师" && echo "公司机F盘" || echo "非公司机F盘"
# 检查G盘是否存在（家用机）
test -d "G:/锅师" && echo "家用机G盘" || echo "非家用机G盘"
```

### 周文件夹计算规则

```python
# 给定产出日期，计算所属周文件夹
from datetime import datetime, timedelta

def calc_week_folder(date_str):
    """输入YYYY-MM-DD，返回MMDD-MMDD周文件夹名"""
    date = datetime.strptime(date_str, "%Y-%m-%d")
    # 周一=0, 周日=6
    monday = date - timedelta(days=date.weekday())
    sunday = monday + timedelta(days=6)
    return f"{monday.strftime('%m%d')}-{sunday.strftime('%m%d')}"

# 示例：2026-06-17（周三）→ 0615-0621
```
