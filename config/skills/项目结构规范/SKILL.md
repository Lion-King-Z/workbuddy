## 文件命名铁律（从上一次出错教训中凝结）

**所有文件名必须使用下划线 `_`，禁止使用点号 `.` 作为字段分隔符。**

| 文件类型 | 命名格式 | 示例 |
|---------|---------|------|
| 产业链深度研究 | `YYYY-MM-DD_题材名产业链深度研究.md` | `2026-06-14_国产AI芯片产业链深度研究.md` |
| 电话会分析 | `YYYY-MM-DD_电话会_关键词.md` | `2026-06-11_电话会_硅片_CCL电子布.md` |
| 每日简报 | `YYYY-MM-DD_每日简报.md` | `2026-06-13_每日简报.md` |
| 月报 | `YYYY-MM_月报.md` | `2026-06_月报.md` |
| 每周研判 | `YYYYWWw_每周研判.md` | `2026W24_每周研判.md` |

**检查清单（每次新建文件时强制对照）：**
1. ✅ 日期格式：`YYYY-MM-DD`（不是 `YYYY-MM-DD.`、`YYYY.MM.DD`）
2. ✅ 分隔符：只使用 `_`（不是 `.`、`-`、空格）
3. ✅ 文件扩展名：`.md`（小写）
4. **先改好文件名→再同步到IMA**，顺序不可颠倒

## 核心原则

1. **2目录+N文件**：深度研究/ + 浑水调研/ + 根目录独立文件
2. **深度研究 = 产业链**：仅放产业链深度研报，不放电话会、日报等
3. **浑水调研 = 浑水生态**：电话会、景气度扫描、紧缺度分析、月度跟踪——凡浑水相关全进此处
4. **时间容器 > 类型容器**：按产出日期归入周文件夹（MMDD-MMDD/），不是按文件类型分文件夹
5. **跨周期追踪表放根目录**：紧缺度追踪总表、时间线等跨周期文件放根目录，不拆入周文件夹
6. **个人成长在项目外**：`F:\个人成长\` 独立管理，不在锅师项目内
7. **无形式化产出**：不生成日报/简报/研报，只产出电话会深度分析

## 周文件夹规则

- 按产出日期归入对应周
- 周文件夹命名：`MMDD-MMDD`（起止日期）
- 例如 6月15日的产出归入 `0615-0621/`
- 周一~周日为一周

## 路径切换

- 家用机：`G:\锅师\`
- 公司机：`F:\workbuddy\锅师\`
- 个人成长：家用机 `G:\个人成长\`，公司机 `F:\个人成长\`
