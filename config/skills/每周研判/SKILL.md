# 每周研判 Skill

从浑水跟踪月度总账 + 聚宝盆索引中提取过去一周的变化，生成结构化周度报告。

## 核心逻辑

三件事：
1. 浑水跟踪了哪些信号 → 本周变化了什么
2. 聚宝盆新增了哪些研究 → 本周产出了什么
3. 两者交叉 → 什么信号在强化、什么在消退

## 触发条件

- 用户说「本周研判」「周度总结」「一周信号变化」「每周报告」「周报」
- 建议配置自动化：每周日晚 21:00（RRULE: `FREQ=WEEKLY;BYDAY=SU;BYHOUR=21`）

## 工作流

### 第一步：读取信号源

```
1. 读取 电话会分析/浑水跟踪_YYYY-MM.md → 累计信号强度表 + S级信号记录表
2. 读取 聚宝盆索引.md → 筛选最近7天的新增研究记录
3. 读取 IMA 聚宝盆知识库 → 本周新增的电话会分析笔记（可选，用于深度引用）
```

### 第二步：信号变化分析

| 分析维度 | 方法 |
|----------|------|
| **信号新增** | 本周首次出现的行业/环节 |
| **信号强化** | 提及频次上升、紧缺度升级的环节 |
| **信号消退** | 上周有本周无，或紧缺度降级 |
| **S级信号状态** | 新触发 / 持续活跃 / 已消退 |

### 第三步：研究产出统计

| 统计项 | 来源 |
|--------|------|
| 本周产业链研究 | 索引中「产业链」类型的新增条目 |
| 本周电话会分析 | 索引中「电话会」类型的新增条目 |
| 本周个股研究/专题 | 索引中「个股」「专题」类型的新增 |

### 第四步：交叉验证

电话会信号 vs 产业链研究产出 → 有无背离？
- 电话会多次提及但未做深度研究 → 标注「待研究」
- 做了深度研究但电话会未提及 → 标注「独立发现」

### 第五步：输出周报

同步到 IMA 聚宝盆（个股研究文件夹）+ 本地 `宏观分析/YYYY-MM-DD_每周研判.md`

## 存储

| 输出 | 位置 |
|------|------|
| IMA 聚宝盆 | 个股研究文件夹（每周研判笔记） |
| 本地 | `宏观分析/YYYY-MM-DD_每周研判.md` |

## 禁止行为

- 跳过信号源读取直接输出
- 遗漏本周任何一条电话会信号
- 周报写到产业链研究或电话会分析文件夹（必须放宏观分析）
