# 每周研判 Skill v2.0

**定位变更**：v1.0（汇总排名）→ v2.0（拐点识别+操作建议）
日度做"看今天"，周度做"看方向"——不重复紧缺度排名，聚焦本周累加后的方向变化。

## 核心逻辑

不做排名，做三件事：
1. 拐点识别 — 本周信号方向变化：加速恶化 / 观察升级确认 / 预期差最大
2. 交叉验证 — 电话会主观判断 vs 紧缺度日报数据，有无背离
3. 操作建议 — 基于拐点识别，输出可执行的加仓/减仓/提问方向

## 触发条件

- 用户说「本周研判」「周度总结」「一周信号变化」「每周报告」「周报」
- 自动化：每周日晚 21:00（RRULE: `FREQ=WEEKLY;BYDAY=SU;BYHOUR=21`）

## 工作流

### Step 0：聚宝盆缓存引用

搜聚宝盆（KB: `9P9LfmFJRFwLidUrzKQu0cZZ5QMVxtv61QA9iFoV90g=`）本周全部电话会 + 紧缺度日报标题。
提取已有判断→引用不重算。日度已分析过的信号→直接引用结论→只做「本周状态更新」。
目的：周报是信号聚合+状态更新+方向判断，不重新分析。

### Step 1：拐点识别（核心）

读取浑水调研/浑水跟踪_YYYY-MM.md + 本周紧缺度日报，回答三个问题：

**Q1：哪些信号在加速恶化？**
- 本周紧缺度5/5维持未缓解 + 涨价幅度继续扩大 → 恶化方向
- 本周紧缺度从3/5→4/5或4/5→5/5 → 加速恶化
- 输出：恶化方向清单，按紧迫性排序，每个带核心数据和来源引用

**Q2：哪些信号从「观察」升级到「确认」？**
- 电话会S级信号有新的公开验证 → 升级确认
- 正文NLP提取累积一周达到确认阈值 → 升级确认
- 输出：升级信号清单 + 可操作标的

**Q3：哪个方向是本周最大预期差？**
- 市场关注度低 + 实际紧缺度≥4/5 → 正预期差（市场没发现的机会）
- 市场关注度高 + 紧缺度实际在缓解 → 负预期差（过度拥挤的风险）
- 输出：预期差 Top 3

### Step 2：交叉验证

电话会信号 vs 紧缺度日报 → 有无背离？
- 电话会多次提及但紧缺度日报未达标 → 「主观预期 vs 数据滞后」
- 紧缺度日报强信号但电话会未提及 → 「自动化发现，人工待确认」

**信源**：聚宝盆（P0.5缓存）> 浑水调研（P0唯一IMA信源）。

### Step 3：操作建议

基于三个问题的结论：
- **加仓方向**：恶化+确认+正预期差
- **观察/减仓方向**：改善中 / 过度交易的
- **下周电话会优先提问** 3-5个方向

### Step 4：输出

同步到 IMA 聚宝盆（浑水调研/当前周文件夹）+ 本地 `浑水调研/<周文件夹>/YYYYWww_每周研判.md`

## 禁止行为

- 从零生成紧缺排名（日度已做，直接引用）
- 输出「本周研究产出」统计表（与拐点判断无关）
- 输出「数据源：」、「电话会N份」等元数据说明行
- 跳过聚宝盆预查询直接从零分析
- 周报生成后不沉淀到IMA
