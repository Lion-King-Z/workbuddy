# 组长

把一段产业链研报文字拆解成多个品种，每个品种按**流水线顺序**穿透三轮：

```
Phase 0: 文字解析 + 信息丰度评估 → 提取品种清单
  ↓
Phase 1: 瓶颈分析 → 是真稀缺还是炒作？ → 调用 产业链瓶颈投研skill
  ↓
Phase 2: 景气度评估 → 产业走到哪一步了？ → 调用 景气度skill
  ↓
Phase 3: PE估值落地 → 股价反映预期了吗？ → 调用 pe-valuation-analyst
  ↓
Phase 4: 综合推荐矩阵 → 逻辑/确定性/预期差/估值/仓位 → 最终由用户决定
```

输出一份**综合报告**（单MD全品种汇总），**默认不同步 IMA**。不拆成多个独立品种MD，保持研究思路的连贯性。

---

## ⛔ 绝对铁律

**永远不要人工确认，直接跑完。** 用户给指令后：
- 不反问"要不要继续"、"重点突哪个"
- 不评估信息丰度后停下来等补充
- 不分阶段确认（Phase 0→1→2→3→4一口气穿透）
- 信息缺口全部自动WebSearch搜索补齐
- 搜不到就标注"数据不足"继续推进，不中断
- 跑完直接输出结果，不等用户说"继续"

---

## 核心承诺

给定一段产业链研报文字或分析框架（不限于文字，也可从用户的三阶求导/类比框架出发），自动提取核心品种/材料，对每个品种串行跑完「瓶颈验证→产业阶段→估值落地」三步，输出一份结构化综合报告。用户做最终决策。

`用户指令 → 品种提取 → 自动搜索补全信息缺口 → (瓶颈→景气度→PE) × N个品种 → 综合推荐矩阵 → 直接输出`

---

## 默认行为

全自动流水线。用户提供文字后，自动解析→**自动搜索补充信息缺口**→逐个品种穿透→输出综合报告MD。

- ⛔ **绝对不反问、不确认、不暂停**（一口气跑到底）
- 默认**不同步 IMA**（除非用户说"同步"）
- 默认**串行执行**（同品种内三步必须串行；品种间可并行）
- ⛔ **禁止反问用户**：信息缺口一律通过WebSearch自动搜索补齐。搜索不到标注"数据不足"继续推进，绝不中断问用户

## Phase 0：文字解析 + 信息丰度评估

### 0a. 识别材料品种与逻辑

按文中出现的顺序，提取每种核心材料/品种：

| 字段 | 说明 |
|------|------|
| 品种名称 | 如"六氟化钨 (WF₆)" |
| 产业逻辑 | 文中对该品种的核心论述 |
| 潜在A股标的 | 尝试搜索关联上市公司 |
| 是否可研究 | 有标且逻辑自洽=是 / 无标的或逻辑存疑=否 |

### 0b. 上下文提取

- 宏观背景：地缘政治、技术路线、政策导向、资金背书
- 产业链全景：各品种之间的上下游关系
- 关键排除项：文中明确说"排除/短期不看好/替代方案"的品种

### 0c. 品种入池判断

| 条件 | 处理 |
|------|------|
| 有可识别的A股标的 + 逻辑自洽 | ✅ 入池，进入三步穿透 |
| 无A股标的 | ⚠️ 标记"无可投标的"，仅输出逻辑摘要，不跑三步 |
| 文中明确说"排除/短期难商用/替代方案" | ⛔ 筛掉，标记"已排除"，输出一句跳过理由 |

### 0d. 信息缺口自动补全

对每个入池品种，对比已提取信息与Phase 1-3的必答项清单。**存在缺口→自动WebSearch搜索补齐，不向用户提问。**

| 缺口类型 | 补全策略 |
|------|------|
| 品种名称/逻辑有，缺标的 | WebSearch `"{品种} A股 标的 上市公司"` |
| 缺供给结构/壁垒 | WebSearch `"{品种} 产能 竞争格局 CR5 壁垒"` |
| 缺产业阶段/TAM | WebSearch `"{品种} 市场空间 TAM 产业阶段"` |
| 缺估值/一致预期 | WebSearch `"{标的代码} PE 盈利预测 一致预期"` + tdx-connector 实时行情 |
| 搜索不到 | 标注"数据不足，基于公开信息框架性判断"，继续推进 |

🔴 **CHECKPOINT — Phase 0 出口**：品种清单提取完毕+信息缺口已自动补全。若入池品种 = 0，🛑 **STOP**，回推用户"无法提取有效标的，请确认研究文字"。若入池品种 ≥ 1，直接进入 Phase 1。

## 信息补全规则（v2.1 自动搜索代替提问）

### 核心原则

⛔ **不向用户提问**。信息缺口全部通过 WebSearch 自动搜索补全。搜索不到则标注"数据不足"，继续推进不中断。

### 各Phase信息采集后自动搜索策略

**Phase 1 采集不到核心信息时**：
```
WebSearch "{品种} 全球产能 CR5 竞争格局 扩产"
WebSearch "{品种} 涨价 紧缺 供需 2026"
WebSearch "{品种} 国产化率 替代 壁垒"
```

**Phase 2 采集不到核心信息时**：
```
WebSearch "{品种} 市场规模 TAM 2026 2027"
WebSearch "{品种} 产业阶段 生命周期 国产替代进度"
```

**Phase 3 采集不到核心信息时**：
```
TDX-Connector tdx_quotes + tdx_kline 获取实时行情
WebSearch "{标的代码} 盈利预测 EPS 一致预期 2026 2027"
WebSearch "{标的代码} PE 估值 目标价 券商"
```

### 数据不足标注

搜索不到时，在对应Phase标注 `[数据不足]` + 简要说明缺了什么，然后给参考区间而非精确分，继续推进。

## Phase 1：瓶颈分析（验证真稀缺 vs 炒作）

对每个入池品种，调用 `产业链瓶颈投研skill`（Chokepoint Hunter v3.0）前先完成信息采集：

### Phase 1 信息采集清单

| 信息项 | 类型 | 说明 |
|------|------|------|
| 1.1 产业链位置 | 必答 | 该品种在上下游中的位置，上游原料/中游材料/下游应用 |
| 1.2 供给端结构 | 必答 | 全球产能分布、中国占比、CR5、扩产周期、主要厂商 |
| 1.3 核心壁垒 | 必答 | 技术壁垒/资源壁垒/工艺壁垒/认证壁垒/资金壁垒 |
| 1.4 需求端驱动 | 条件 | 下游AI/HBM/服务器/新能源车用量弹性（若原文未明确则问） |
| 1.5 紧缺信号 | 条件 | 价格/产能/客户/库存/投资/跨市场六维中哪些信号存在 |
| 1.6 替代威胁 | 深度 | 是否有替代材料/技术路线？替代难度多大？ |
| 1.7 地缘政治 | 条件 | 是否涉及出口管制、制裁、原料国集中度 |

### Phase 1 调用下游Skill

信息缺口通过WebSearch自动补齐后，直接调用 `产业链瓶颈投研skill`：

```
输入：品种名 + 产业链位置 + 供给结构 + 壁垒 + 需求驱动 + 紧缺信号 + 替代/地缘信息
输出：瓶颈分 B（0-5）+ 卡脖子节点（N1-N8）+ 真假稀缺判定 + 认知偏差分 + 信源[T1/T2/T3]
```

**核心输出**：
- 瓶颈分 B（0-5）
- 卡脖子节点识别（N1-N8标号）
- 真假稀缺判定：是真供给刚性还是短期炒作？
- 单点依赖节点
- 信源标注：[T1]/[T2]/[T3]

## Phase 2：景气度评估（判定产业发展进度）

对每个入池品种，调用 `景气度skill` 前先完成信息采集：

### Phase 2 信息采集清单

| 信息项 | 类型 | 说明 |
|------|------|------|
| 2.1 产业阶段 | 必答 | 0-6级中当前处于哪一级（若不明显则问） |
| 2.2 TAM/SAM/SOM | 必答 | 全球市场空间、中国市场空间、标的公司可触达空间 |
| 2.3 周期位置 | 必答 | 当前处于上行/下行/拐点/平台期 |
| 2.4 订单/产能可见度 | 条件 | 订单排期、产能利用率、扩产计划、客户认证进度 |
| 2.5 竞争格局 | 条件 | 龙头/二线/新进入者动态，市场份额变化 |
| 2.6 政策与地缘 | 深度 | 补贴、出口管制、关税、国产替代政策影响 |

### Phase 2 调用下游Skill

信息缺口通过WebSearch自动补齐后，直接调用 `景气度skill`：

```
输入：品种名 + 产业阶段 + TAM + 周期位置 + 订单可见度 + 竞争格局 + 政策地缘
输出：景气度分 P（0-5）+ 产业阶段（0-6级）+ TAM与10倍市值检验 + 黄金区间定位
```

**核心输出**：
- 景气度分 P（0-5）
- 产业阶段（0-6级）
- TAM与10倍市值检验
- 黄金区间定位（0.5级→3.5级）
- 仓位配置建议（参考）

## Phase 3：PE估值落地（股价反映预期了吗）

对每个入池品种，调用 `pe-valuation-analyst` v1.1 前先完成信息采集：

### Phase 3 信息采集清单

| 信息项 | 类型 | 说明 |
|------|------|------|
| 3.1 核心标的 | 必答 | A股/港股代码与名称（Phase 0已识别则确认） |
| 3.2 当前估值 | 必答 | 当前市值、PE/PEG/PS、行业百分位 |
| 3.3 一致预期 | 必答 | 2025-2028 EPS/营收预测、增速 |
| 3.4 可比公司 | 条件 | 国内外可比公司及其PE中位数 |
| 3.5 消息催化 | 条件 | 近期已发生/待发生催化事件，每条给置信度 |
| 3.6 风险因素 | 条件 | 订单不及预期、价格下跌、竞争加剧、政策变化 |
| 3.7 历史估值 | 深度 | 标的近3-5年PE中枢与极值 |

### Phase 3 调用下游Skill

信息缺口通过WebSearch+tdx-connector自动补齐后，直接调用 `pe-valuation-analyst`：

```
输入：标的代码 + 产业链故事 + 当前估值 + 一致预期 + 可比公司 + 催化事件 + 风险因素
输出：PE消化分 E（= E_base × 60% + E_catalyst × 40%）+ 基准估值 + 消息催化增量 + 估值弹性分解 + 锚帆比 + 当前定价程度
```

**核心输出**：
- PE消化分 E（= E_base × 60% + E_catalyst × 40%）
- 基准估值（纯基本面，不含消息催化）
- 消息催化增量（概率加权后）
- 估值弹性分解（基准弹性 + 催化弹性 = 综合弹性）
- 锚帆比（基准弹性 / 消息催化弹性）
- 当前定价程度：未定价/半定价/充分定价/透支

## Phase 4：综合推荐矩阵

每个品种输出一张推荐卡（放在该品种 MD 文件末尾）：

| 维度 | 内容 |
|------|------|
| 推荐逻辑 | 一句话，完整逻辑链 |
| 瓶颈分 | B/5（信源：T1/T2/T3） |
| 景气度分 | P/5（产业阶段X级，TAM XX亿） |
| PE消化分 | E/5（E_base X + E_catalyst X） |
| 确定性 | 高/中/低（理由一句话） |
| 预期差 | 市场定价 vs 产业现实的gap |
| 当前估值 | 基准弹性 +XX%，消息催化弹性 +XX%，综合弹性 +XX% |
| 仓位建议 | X%（仅供参考，最终由用户决定） |

### 标的级三情景分析

每个推荐标的额外输出一张**三情景表**，展示当前股价下三种看法：

| 标的（代码） | 现价 | 乐观 | 中性 | 悲观 |
|:---|:---:|:---|:---|:---|
| {名称}（{代码}） | {XX}元 | **{XX}元**（{+XX%}）— {理由} | **{XX}元**（{±XX%}）— {理由} | **{XX}元**（{-XX%}）— {理由} |

- **乐观**：所有消息催化兑现 + 景气度推进 + 瓶颈红利释放。对应 Phase 3 的乐观情景估值。
- **中性**：概率加权后的综合估值。大部分催化按现状兑现。
- **悲观**：纯基准估值。消息催化全部不兑现或证伪，只剩基本面支撑。

每格给出具体价格和一句理由，三情景基于 Phase 3 的四档风险情景精简映射：

| 三情景 | 映射 Phase 3 风险情景 |
|---------|---------------------|
| 乐观 | 乐观（基准+全部催化） |
| 中性 | 中性（基准+概率加权催化） |
| 悲观 | 悲观/谨慎（纯基准/仅高置信度催化） |

🛑 **STOP — Phase 4 出口**：所有品种推荐卡片+三情景表已输出。**不做最终买卖决策**，呈现完整推荐矩阵，等用户拍板。

所有品种跑完后，输出一份**汇总索引表**（单独 MD），列出每个品种的核心指标 + 三情景价格对比。

## 失败模式处理

| 情况 | 处理方式 |
|------|---------|
| 品种无A股纯正标的 | 标"无可投标的"，仅输出逻辑摘要，不跑Phase 1-3 |
| 标的PE估值数据严重不足（新股/亏损/无一致预期） | PE消化分标"数据不足"，仅跑Phase 1+Phase 2，Phase 3给出框架性判断 |
| 瓶颈分析关键信源缺失 | 瓶颈分给参考区间（如"3-4分"），标"待验证：缺XX信源" |
| Phase 2 产业阶段无法判定（数据不足） | 标"无法判定"，降级为基于公开信息的主观推断 |
| Phase 1/2/3 结果严重矛盾（如B=5但P=1） | 🔴 **CHECKPOINT**：标"矛盾信号"，在推荐卡里写清矛盾点，不做强行综合。🛑 **STOP**，把矛盾点呈现给用户，等用户判断是否继续 |
| 用户提供的文字无法解析出任何有效品种 | 回推给用户：无法提取有效标的，请确认研究文字 |
| 搜索不到关键数据 | 降级为"框架性研究"，所有评分给区间而非精确值，明确标注"信息不足，结论供参考" |

## 与三个基础Skill的衔接

本Skill是**串联编排层 + 信息采集层**，不替代三个基础Skill的执行：

| Phase | 基础Skill | 本层职责 | 下游输入 | 下游输出 |
|:---|:---|:---|:---|:---|
| Phase 1 | 产业链瓶颈投研skill | 评估信息丰度、按品种情况提问补全 | 品种+产业逻辑+供给结构+壁垒+需求 | 瓶颈分B + 卡脖子节点 |
| Phase 2 | 景气度skill | 评估信息丰度、按品种情况提问补全 | 品种+产业阶段+TAM+周期+订单 | 景气度分P + 产业阶段 |
| Phase 3 | pe-valuation-analyst | 评估信息丰度、按品种情况提问补全 | 标的代码+产业链故事+估值+预期+催化 | PE消化分E + 估值弹性分解 |
| Phase 4 | 本Skill（综合） | 汇总B+P+E | B+P+E | 推荐矩阵 |

**执行顺序**：Phase 1 → Phase 2 → Phase 3，按顺序串行。Phase 2 引用 Phase 1 的瓶颈结论，Phase 3 引用 Phase 2 的景气度结论。

## 禁止行为

| # | 禁止项 | 原因 | 正确做法 |
|---|--------|------|----------|
| 1 | 不跑Phase 1直接跳Phase 3 | 没瓶颈验证就去估值=空中楼阁 | 必须串行 |
| 2 | 替用户做最终买卖决策 | 用户明确要求"最后还是让我直接决定" | 给出分析+建议，不拍板 |
| 3 | 同品种Phase间并行 | Phase间有逻辑依赖 | 同品种内串行，品种间可并行 |
| 4 | 失败模式发生时强行出分 | 数据不足硬给分=误导 | 诚实标"数据不足" |
| 5 | 默认同步IMA | 用户明确要求"先不同步IMA" | 默认不同步，等用户说同步 |
| 6 | Phase 4 评分取代用户判断 | Phase 4是参考矩阵不是上帝指令 | 明确标注"最终操作由用户决定" |
| 7 | 信息缺口向用户提问 | 用户要求全自动不反问 | 自动WebSearch搜索补齐，搜不到标注"数据不足" |
| 8 | 过度提问 | 用户文字已包含的信息重复问 | 先提取，只搜缺项 |
| 9 | 忽视品种差异 | 不同品种复杂度不同 | 复杂度高的搜更多关键词，简单的搜基础信息即可 |
| 10 | 反问用户 | 用户指令就是全部上下文 | 从用户指令中提取框架，自动搜索补齐细节 |
| 11 | 人工确认/分阶段暂停/问"要不要继续" | 绝对铁律：一口气跑完 | 用户给指令→直接跑→直接输出结果 |
